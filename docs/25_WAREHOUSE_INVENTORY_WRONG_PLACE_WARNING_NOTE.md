# 25.x. Объект найден не на своём месте при инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_CONTINUOUS_FOUND_SCAN_NOTE.md` и фиксирует решение 10.83.

## Решение

Если при инвентаризации через мобильный сканер объект найден не в той зоне, не на том стеллаже или не на той полке, система должна просто показать предупреждение и правильное место хранения.

Система не должна автоматически менять место хранения.

## Логика

Пользователь сканирует объект в рамках инвентаризации выбранной области.

Если система видит, что объект должен храниться в другом месте, она показывает:

- предупреждение;
- текущее ожидаемое место хранения;
- куда объект нужно переложить;
- фото места хранения, если оно есть;
- кнопку открытия карточки или плана, если нужно.

## Что делает пользователь

Техник или сотрудник вручную перекладывает объект на правильное место.

После этого объект может быть отмечен как найденный в рамках инвентаризации.

## Почему так

В этом сценарии задача не обновить базу под фактический беспорядок, а помочь вернуть объект туда, где он должен лежать.

Так склад остаётся организованным, а инвентаризация помогает исправлять расстановку.

## MVP

Для MVP достаточно:

1. показать предупреждение `Объект не из этой зоны`;
2. показать правильный путь хранения;
3. показать фото места, если есть;
4. не менять место хранения автоматически.
