# 25.x. Настраиваемые префиксы категорий

Этот документ дополняет `docs/25_WAREHOUSE_INTERNAL_CODE_FORMAT_NOTE.md` и фиксирует решение 10.65.

## Решение

Префиксы категорий для внутренних кодов складских позиций должны настраиваться в админке.

Компания должна иметь возможность сама задать понятные сокращения для своих категорий.

## Примеры

```text
TRS = фермы
SND = звук
LGT = свет
VID = видео
LED = LED-экраны
STG = сцены
PWR = питание
CBL = кабели / коммутация
```

## Логика

Префикс хранится в настройках категории.

При создании новой складской позиции система берёт префикс категории и генерирует внутренний код по выбранному формату.

Например:

```text
SND-000245
LED-000044
TRS-000123
```

## Правила

Префикс должен быть уникальным внутри компании.

Префикс желательно делать коротким и понятным.

Если префикс категории изменили, уже созданные внутренние коды позиций не должны автоматически меняться без отдельного осознанного действия.

## MVP

Для MVP достаточно дать администратору возможность:

- задать префикс категории;
- изменить префикс категории;
- видеть предупреждение, если такой префикс уже используется;
- использовать префикс при автогенерации новых кодов.
