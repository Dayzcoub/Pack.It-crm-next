# 25 Warehouse — Inventory Schedule Create From Cancelled Link Note

## 10.106. Связь новой инвентаризации со старой отменённой

Решение: при создании новой плановой инвентаризации на основе отменённой не нужно делать заметную пользовательскую связь со старой проверкой.

Причина:
- слишком подробное отображение цепочек `создано на основе`, `отменено`, `пересоздано` может перегружать рабочий интерфейс;
- PACK.IT должен помогать складу работать быстрее, а не акцентировать внимание на каждом прошлом управленческом действии;
- особенно для маленьких команд и живых прокатных процессов лишняя детализация может выглядеть как ненужная бюрократия.

Правило:
- отменённая проверка остаётся в истории со статусом `отменено`;
- новая проверка создаётся как самостоятельная плановая инвентаризация;
- в рабочей карточке новой проверки не нужно явно показывать фразу вроде `создано на основе отменённой проверки от 15 июля`;
- пользователь видит актуальную задачу, дату, область, ответственного и рабочие параметры без лишнего акцента на прошлой отмене.

Допустимый технический след:
- система может хранить внутренний технический reference/source id, если это понадобится для отладки или миграций;
- такой след не должен быть основным пользовательским элементом и не должен отображаться как контрольный ярлык;
- для пользователя важнее текущая задача, а не подробная цепочка прошлых действий.

Архитектурный смысл:
- новая инвентаризация на основе отменённой — это удобное копирование параметров, а не восстановление старой задачи;
- история должна быть достаточной для прозрачности, но не чрезмерной в ежедневном интерфейсе;
- продукт должен поддерживать и маленькие гибкие команды, где лишняя бюрократия мешает работе.

MVP-подход:
- создать новую проверку как независимую сущность;
- не показывать явную связь со старой отменённой проверкой в основной карточке;
- сохранить только минимально необходимую системную историю отменённой проверки.
