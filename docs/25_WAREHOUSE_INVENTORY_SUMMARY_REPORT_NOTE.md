# 25.x. Итоговый отчёт после инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_AUDIT_NOTE.md` и фиксирует решение 10.90.

## Решение

После завершения инвентаризации нужно показывать небольшой итоговый отчёт.

Отчёт должен быстро показать результат проверки и дать переход к проблемным строкам.

## Что показывать

В отчёте можно показывать:

- сколько позиций ожидалось к проверке;
- сколько проверено;
- сколько найдено;
- сколько не найдено;
- сколько найдено не на своём месте;
- сколько создано проблемных строк;
- кто проводил проверку;
- дата и время завершения;
- область проверки: склад, зона, стеллаж, полка или категория.

## Пример

```text
Проверено: 142 позиции
Найдено: 137
Не найдено: 3
Не на своём месте: 2
Создано проблемных строк: 5
```

## Логика

Итоговый отчёт не должен быть большим финансовым или аналитическим отчётом.

Это короткая сводка результата инвентаризации.

После просмотра отчёта пользователь может перейти к списку проблемных строк.

## MVP

Для MVP достаточно:

1. показать компактную сводку после завершения проверки;
2. показать количество проблемных строк;
3. дать кнопку `Перейти к проблемам`;
4. сохранить итог в истории инвентаризации.
