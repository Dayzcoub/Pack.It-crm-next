# 25 Warehouse — Temporary Process Rules Period Note

## 10.124. Период действия временного правила склада

Решение: временное правило склада должно задаваться гибко через период действия и возможность досрочного отключения.

Правило:
- администратор задаёт дату начала действия временного правила;
- администратор задаёт дату окончания действия временного правила;
- временное правило действует в период `с` / `по`;
- администратор может отключить временное правило вручную до завершения периода;
- после окончания периода правило автоматически возвращается к исходному значению;
- при досрочном отключении правило также возвращается к исходному значению.

Сценарии:
- строгий возврат на сезон;
- усиленный контроль на время переезда склада;
- временное правило на период крупного проекта;
- тестирование нового процесса в ограниченный период.

Что хранить:
- process key;
- rule key;
- original value;
- temporary value;
- start date/time;
- end date/time;
- enabled/disabled status;
- disabled manually flag;
- disabled by;
- disabled at;
- admin comment/reason.

Архитектурный смысл:
- временные правила не должны зависать навсегда;
- период делает поведение предсказуемым;
- досрочное отключение нужно, если временное правило оказалось больше не актуально или мешает работе.

MVP-подход:
- требовать период `с` и `по` при создании временного правила;
- разрешить ручное отключение до даты окончания;
- предусмотреть автоматическое возвращение исходного значения после окончания периода.
