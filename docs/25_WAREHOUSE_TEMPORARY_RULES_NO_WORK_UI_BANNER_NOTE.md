# 25 Warehouse — Temporary Rules No Work UI Banner Note

## 10.125. Отображение временных правил в рабочем интерфейсе склада

Решение: активные временные правила не нужно показывать в обычном рабочем интерфейсе склада.

Правило:
- не выводить плашки вроде `до 30 сентября действует строгий режим возврата` в ежедневных рабочих экранах;
- оператор склада просто работает по текущим правилам системы;
- временные правила видны в административных настройках склада;
- директор и администратор могут видеть список активных временных правил, сроки и причины;
- обычный пользователь не должен получать лишний визуальный шум.

Причина:
- рабочий интерфейс должен оставаться чистым;
- если временное правило активно, система сама применяет нужные требования;
- пользователю важны конкретные действия в текущем процессе, а не административная причина режима.

MVP-подход:
- не добавлять рабочие баннеры временных правил;
- показывать временные правила только в настройках склада и, при необходимости, в audit/admin-разделе;
- в рабочем процессе отображать только конкретные требования, если они нужны: обязательный комментарий, подтверждение, дедлайн и т.д.
