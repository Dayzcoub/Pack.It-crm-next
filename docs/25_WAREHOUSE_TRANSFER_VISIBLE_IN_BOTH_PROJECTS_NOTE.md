# 25 Warehouse — Transfer Visible In Both Projects Note

## 10.146. Transfer-блок в обоих проектах

Решение: transfer-блок должен отображаться в сборочных листах обоих связанных проектов.

Правило:
- в проекте-источнике transfer-блок показывает, что после работы/демонтажа оборудование нужно передать или увезти на другой проект;
- в проекте-получателе transfer-блок показывает, что часть оборудования приезжает не со склада, а с другой площадки;
- оба проекта должны видеть одну и ту же transfer-связь с разных сторон;
- это помогает командам на обеих площадках понимать цепочку движения оборудования.

Как отображать:
- source project: `передать на проект B`;
- target project: `получить с проекта A`;
- оборудование / комплект;
- количество;
- время забора;
- время доставки;
- ответственный;
- предупреждение по времени, если есть;
- комментарий.

Назначение:
- не терять transfer в документах;
- избежать ситуации, когда первая площадка не знает, что комплект нужно отдать;
- избежать ситуации, когда вторая площадка ждёт оборудование со склада, хотя оно едет с другого проекта.

MVP-подход:
- хранить одну transfer-сущность между двумя проектами;
- показывать её в сборочном листе обоих проектов;
- не создавать отдельный transfer PDF на MVP;
- PDF/print версия сборочного листа каждого проекта может включать свой transfer-блок.
