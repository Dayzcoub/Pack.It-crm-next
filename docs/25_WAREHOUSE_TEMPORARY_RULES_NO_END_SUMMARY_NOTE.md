# 25 Warehouse — Temporary Rules No End Summary Note

## 10.131. Итоговая запись после окончания временного правила

Решение: отдельную итоговую запись после окончания временного правила склада делать не нужно.

Правило:
- не создавать отдельный summary/event после автоматического завершения временного правила;
- достаточно хранить статус самого временного правила;
- в карточке или истории временного правила может быть видно, что оно завершено автоматически или отключено вручную;
- рабочий интерфейс склада не перегружается дополнительными итоговыми сообщениями.

Причина:
- временное правило уже имеет период действия, статус и историю изменения;
- отдельная итоговая запись дублирует эту информацию;
- для MVP достаточно простой модели статусов.

MVP-подход:
- хранить status: scheduled / active / finished / disabled;
- хранить finished_at или disabled_at при необходимости;
- не создавать отдельный итоговый audit item при штатном завершении периода.
