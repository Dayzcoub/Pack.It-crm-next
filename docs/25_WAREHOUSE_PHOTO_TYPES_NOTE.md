# 25.x. Типы складских фото

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.29.

## Решение

Для складских фото нужно заложить классификацию по типам, но заполнение типов фото не должно быть обязательным для каждой операции.

Типы фото нужны как удобная структура для поиска, истории и разбора спорных ситуаций, но не должны превращать складскую работу в бюрократию.

На старте можно предусмотреть типы:

- фото карточки позиции;
- фото экземпляра;
- фото при выдаче;
- фото при возврате;
- фото повреждения / проблемы;
- другое фото.

Фото может быть добавлено без типа, если пользователь быстро прикрепляет материал в процессе работы.

Если тип выбран, система может использовать его для фильтрации, истории экземпляра, документов выдачи/возврата и проблемных строк.

Для MVP типизация фото является опциональной возможностью, а не обязательным требованием.
