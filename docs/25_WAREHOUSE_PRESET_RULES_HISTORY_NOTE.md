# 25 Warehouse — Preset Rules History Note

## 10.115. История применения пресета правил склада

Решение: после применения пресета настроек склада нужно сохранять историю изменения правил.

Что фиксировать:
- кто применил пресет;
- дата и время действия;
- название выбранного пресета;
- список процессов, для которых были обновлены правила;
- значения правил до применения пресета;
- значения правил после применения пресета.

Правило:
- история нужна для административной прозрачности;
- она не должна перегружать ежедневный складской интерфейс;
- обычному складскому пользователю не нужно видеть подробную историю настроек в рабочем процессе;
- директор и администратор должны иметь доступ к истории в настройках склада или audit-разделе.

Архитектурный смысл:
- применение пресета меняет поведение складских процессов;
- поэтому изменение должно быть отслеживаемым;
- если после применения пресета появились вопросы по правилам, администратор может понять, когда и почему они изменились.

MVP-подход:
- сохранять событие применения пресета в истории настроек склада;
- хранить минимум: пользователь, время, пресет, список затронутых процессов;
- подробные старые/новые значения можно хранить как JSON diff или отдельный объект истории;
- не выводить эту историю в обычные рабочие карточки склада.
