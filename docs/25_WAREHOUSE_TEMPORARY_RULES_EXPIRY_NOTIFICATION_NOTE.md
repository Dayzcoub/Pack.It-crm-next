# 25 Warehouse — Temporary Rules Expiry Notification Note

## 10.126. Уведомление перед окончанием временного правила склада

Решение: перед окончанием временного правила склада нужно уведомлять администратора и/или директора.

Сценарий:
- администратор включил временное правило на период `с` / `по`;
- до окончания периода система отправляет уведомление;
- администратор заранее понимает, что временное правило скоро отключится и процесс вернётся к обычным настройкам.

Правило:
- уведомление отправляется только пользователям с административными правами;
- обычных складских пользователей уведомлением не перегружаем;
- уведомление должно указывать процесс, правило и дату окончания;
- администратор может заранее продлить период, отключить правило вручную или ничего не делать.

Рекомендуемый MVP-вариант:
- уведомлять за 1 день до окончания временного правила;
- дополнительно можно показывать активные временные правила в административных настройках склада;
- после окончания периода правило возвращается к исходному значению.

Архитектурный смысл:
- окончание временного правила не должно быть неожиданным для управляющих пользователей;
- при необходимости администратор может продлить временный режим до его завершения;
- рабочий интерфейс склада остаётся чистым, без лишних плашек для операторов.
