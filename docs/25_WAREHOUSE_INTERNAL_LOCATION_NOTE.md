# 25.x. Внутренние перемещения и место хранения

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.32.

## Решение

На старте не нужен отдельный журнал внутренних перемещений между местами хранения.

Для MVP достаточно просто менять текущее место хранения в карточке складской позиции или экземпляра.

Примеры изменения места хранения:

- склад;
- офис;
- машина;
- другая локация.

История внутренних перемещений может быть добавлена позже, если складской учёт станет более строгим или появится несколько крупных складских зон.

Главная задача MVP — быстро понимать, где сейчас находится позиция или экземпляр, без лишней операционной бюрократии.
