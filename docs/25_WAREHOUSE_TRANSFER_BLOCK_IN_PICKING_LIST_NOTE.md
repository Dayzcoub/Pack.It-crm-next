# 25 Warehouse — Transfer Block In Picking List Note

## 10.145. Transfer-блок внутри сборочного листа

Решение: для переезда оборудования между проектами достаточно отдельного блока внутри сборочного листа. Отдельный PDF/документ transfer на MVP не нужен.

Правило:
- transfer between projects отображается отдельным блоком в сборочном листе;
- отдельный самостоятельный transfer-документ на старте не делаем;
- блок должен быть связан с проектом-источником и проектом-получателем;
- блок должен показывать, какое оборудование переезжает, откуда, куда и кто ответственный.

Что показывать в transfer-блоке:
- оборудование / комплект;
- количество;
- проект или площадка, откуда забрать;
- проект или площадка, куда доставить;
- плановое время забора;
- плановое время доставки;
- ответственный;
- предупреждение, если между проектами мало времени;
- комментарий.

Назначение:
- не плодить лишние документы;
- оставить сборочный лист главным рабочим документом склада и команды;
- сделать переезды видимыми прямо в контексте проекта.

MVP-подход:
- transfer rows live inside picking list as a separate section;
- no separate transfer PDF in MVP;
- PDF/print version of picking list may include the transfer block;
- interactive picking list shows transfer block with responsible and warning status.
