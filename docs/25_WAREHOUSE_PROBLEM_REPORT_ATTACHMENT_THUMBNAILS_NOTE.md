# 25 Warehouse — Problem Report Attachment Thumbnails Note

## 10.193. Attachment thumbnails in the internal problem/loss report

Decision: the internal warehouse problem/loss report should include photos and files as small thumbnails where possible.

Rules:
- show small thumbnails for photo attachments;
- show compact file cards or file icons for non-image files;
- attach previews to the related problem row or section;
- if multiple problems are combined into one report, keep attachments grouped by problem;
- if no attachments exist, do not show an empty attachment area.

Purpose:
- make the problem easier to understand directly from the report;
- keep evidence visible without forcing users to open separate records;
- support reports for damage, missing items, incomplete returns, and similar warehouse cases.

MVP approach:
- include small image previews where available;
- include compact file references for non-image attachments;
- keep full files available through stored attachment links in the system;
- keep the report internal by default.
