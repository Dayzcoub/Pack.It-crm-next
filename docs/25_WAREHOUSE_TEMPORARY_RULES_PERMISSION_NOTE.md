# 25 Warehouse — Temporary Rules Permission Note

## 10.129. Права на управление временными правилами склада

Решение: управление временными правилами склада должно быть отдельным разрешением, которое выдаёт администратор.

Правило:
- по умолчанию временными правилами управляет администратор или директор;
- если нужно, администратор может выдать это право заведующему складом, старшему технику или другой роли;
- доступ должен определяться не только названием роли, а конкретным permission;
- без разрешения пользователь не может создавать, изменять, отключать или продлевать временные правила.

Возможное permission:
- `warehouse.process_rules.temporary.manage`

Что входит в разрешение:
- создать временное правило;
- изменить период действия;
- отключить временное правило раньше срока;
- видеть список активных временных правил;
- видеть историю изменений временного правила.

Архитектурный смысл:
- разные компании по-разному распределяют ответственность;
- маленький прокат может оставить всё на администраторе;
- большая команда может передать управление временными режимами старшему складскому сотруднику;
- при этом доступ остаётся контролируемым.

MVP-подход:
- заложить отдельное permission;
- по умолчанию включить для admin/director;
- разрешить администратору выдавать permission другим ролям при необходимости.
