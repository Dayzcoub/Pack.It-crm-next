# 25.x. Мобильный режим сканера

Этот документ дополняет `docs/25_WAREHOUSE_SCAN_QUICK_ACTIONS_NOTE.md` и фиксирует решение 10.73.

## Решение

В системе нужен отдельный мобильный режим сканера.

Это будет отдельный экран или режим интерфейса, который позволяет быстро сканировать QR / штрихкоды с телефона и выполнять складские действия без лишней навигации.

## Зачем это нужно

Мобильный сканер полезен для:

- сборки оборудования;
- выдачи;
- возврата;
- инвентаризации;
- поиска позиции на складе;
- фиксации проблем;
- проверки кейсов и контейнеров;
- работы на складе без компьютера.

## Логика

Пользователь открывает режим `Сканер` на телефоне.

После сканирования система определяет объект и показывает быстрое меню действий с учётом роли пользователя и текущего контекста.

Примеры действий:

- открыть карточку;
- отметить в сборочном листе;
- выдать;
- принять возврат;
- отметить на инвентаризации;
- зафиксировать проблему;
- открыть место хранения.

## Права доступа

Мобильный режим сканера не должен обходить права доступа.

Пользователь видит только те действия, которые разрешены его ролью и настройками компании.

## MVP

Для MVP достаточно:

1. отдельный экран `Сканер`;
2. сканирование QR / штрихкода камерой телефона;
3. распознавание объекта;
4. быстрое меню действий;
5. переход в карточку объекта или текущую складскую операцию.
