# 25.x. Публичное название складской позиции

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.44.

## Решение

Для складских позиций не нужно отдельное публичное название для клиента.

В MVP используем основное название позиции из базы.

Причины:

- не нужно плодить дополнительные поля;
- не нужно делать множество ручных замен в базе;
- складской каталог остаётся проще;
- смета и документы могут использовать одно название позиции.

Если позиция имеет техническое или непонятное название, это допустимо.

В будущем, если появится отдельный клиентский каталог или режим красивых коммерческих предложений, публичные названия можно добавить отдельной настройкой, но в базовой складской модели они не нужны.
