# 25 Warehouse — Return Act Detailing Stop Note

## 10.180. Stop detailing the return act

Decision: the return act has enough detail for the current warehouse documentation stage.

Already fixed:
- the return act uses the same settings approach as the handout act;
- it is an internal warehouse document;
- problem rows are shown as a separate block only when they exist;
- the physical delivery person is recorded;
- external delivery person fields include name, phone, and comment;
- the accepted-by warehouse user is recorded;
- per-item full or partial accepted statuses are not required by default;
- a compact return summary is shown;
- problem-row attachments are included where available;
- attachments are shown as small thumbnails where possible;
- generated return acts have history and versions;
- older versions are marked outdated;
- outdated reason can be automatic and manually edited.

Decision on PDF/print detail:
- do not start a separate deep discussion for print/PDF layout now;
- reuse the same document principles as the handout act;
- keep the specification sufficient for MVP.

Next documentation area:
- move to the inventory report or warehouse problem/loss report.
