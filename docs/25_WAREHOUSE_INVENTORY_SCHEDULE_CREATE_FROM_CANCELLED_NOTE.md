# 25 Warehouse — Inventory Schedule Create From Cancelled Note

## 10.105. Создание новой плановой инвентаризации на основе отменённой

Решение: нужно разрешить действие `создать новую на основе отменённой`.

При этом отменённая плановая инвентаризация не восстанавливается и не возвращается в активный календарь. Система создаёт новую плановую инвентаризацию как отдельную сущность.

Пример сценария:
- директор отменил плановую проверку;
- позже стало понятно, что похожая проверка всё-таки нужна;
- директор нажимает `создать новую на основе этой`;
- система копирует основные параметры отменённой проверки;
- пользователь выбирает новую дату и при необходимости корректирует параметры;
- создаётся новая календарная задача и новая плановая инвентаризация.

Что можно копировать из отменённой проверки:
- шаблон инвентаризации;
- область проверки;
- категории / фильтры;
- ответственного по умолчанию;
- дополнительные настройки проверки, если они были заданы.

Что нельзя копировать как готовое состояние:
- статус `отменено`;
- историю отмены старой проверки;
- старую дату как актуальную без подтверждения;
- результаты, проблемные строки и отчёты, если они уже существовали в старой проверке.

Правило:
- новая проверка должна иметь собственный жизненный цикл, календарную задачу и историю;
- старая отменённая проверка остаётся в истории со статусом `отменено`;
- в истории новой проверки можно сохранить ссылку на источник: `создано на основе отменённой проверки`.

Архитектурный смысл:
- это ускоряет повторное создание похожих плановых проверок;
- не ломает аудит отменённых задач;
- не создаёт путаницу между восстановлением старой проверки и созданием новой.

MVP-подход:
- действие доступно директору/администратору;
- при создании новой проверки обязательно выбрать новую дату;
- скопированные параметры можно отредактировать перед сохранением.
