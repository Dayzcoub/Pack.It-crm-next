# 25 Warehouse — Return Act Summary Note

## 10.175. Summary in the return act

Decision: the return act should include a short summary.

Rules:
- show total positions expected for return;
- show how many returned without problems;
- show whether problem rows exist;
- show count of rows that require review or follow-up;
- keep the summary compact.

Purpose:
- make the return result visible at a glance;
- help managers and warehouse users quickly understand whether the return is clean;
- avoid reading the whole document to find exceptions.

MVP approach:
- add a compact summary block near the top of the return act;
- include problem count only when problem rows exist;
- keep the return act internal;
- detailed handling remains in the warehouse problem flow.
