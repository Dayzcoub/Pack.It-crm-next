# 25 Warehouse — Inventory Schedule Cancel Reason Note

## 10.103. Причина отмены плановой инвентаризации

Решение: при полной отмене плановой инвентаризации директор или администратор должен оставить комментарий с причиной отмены.

Пример сценария:
- плановая проверка была создана по ошибке;
- проверка больше не актуальна;
- проверка заменена другой инвентаризацией;
- директор или администратор отменяет проверку;
- при сохранении отмены система требует комментарий.

Примеры комментариев:
- создана по ошибке;
- проверка больше не актуальна;
- заменена другой инвентаризацией;
- изменилась зона ответственности;
- складская область временно не используется;
- ручное решение директора.

Правило:
- комментарий обязателен именно при отмене обязательной плановой инвентаризации;
- комментарий сохраняется в истории отменённой проверки и в связанной календарной задаче;
- системная история также сохраняет пользователя, дату/время отмены, исходную дату проверки, шаблон/область и ответственного;
- обычный пользователь не отменяет обязательную плановую проверку без соответствующих прав.

Архитектурный смысл:
- отмена плановой проверки должна быть объяснимой и отслеживаемой;
- отменённая проверка не должна исчезать из истории и не должна выглядеть как выполненная;
- причина отмены помогает отличать ошибочно созданные задачи от управленчески отменённых или заменённых проверок.

MVP-подход:
- при переводе плановой инвентаризации в статус `отменено` требовать текстовый комментарий;
- показывать причину отмены в карточке календарной задачи и истории инвентаризаций;
- не считать отменённую проверку просроченной или выполненной.
