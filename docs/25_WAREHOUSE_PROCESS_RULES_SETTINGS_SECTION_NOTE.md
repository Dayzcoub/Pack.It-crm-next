# 25 Warehouse — Process Rules Settings Section Note

## 10.132. Отдельный раздел настроек правил процессов склада

Решение: правила процессов склада должны жить в отдельном разделе настроек склада.

Раздел:
- `Правила процессов`.

Что собрать в разделе:
- строгость процессов;
- пресеты правил склада;
- ручное применение пресета;
- preview изменений перед применением пресета;
- откат последнего применённого пресета;
- временные правила;
- история ручных изменений правил;
- история применения пресетов;
- история изменения временных правил.

Правило:
- управление правилами не должно быть размазано по разным экранам склада;
- администратору должно быть проще понимать, почему склад ведёт себя гибко или строго;
- рабочие экраны выдачи, возврата и инвентаризации не перегружаются настройками.

Архитектурный смысл:
- правила процессов влияют на поведение всего складского модуля;
- их нужно управлять централизованно;
- отдельный раздел помогает отделить ежедневную работу склада от административной настройки процессов.

MVP-подход:
- добавить в настройки склада раздел `Process Rules` / `Правила процессов`;
- внутри сделать блоки: текущие правила, пресеты, временные правила, история;
- доступ к разделу ограничить правами администратора/директора и отдельными permissions.
