# 25 Warehouse — First Run Setup Wizard Note

## 10.111. Мастер первой настройки склада

Решение: при первой настройке склада нужно показывать простой мастер выбора стартового сценария.

Основная задача мастера:
- помочь администратору быстро выбрать начальный тип складского процесса;
- применить подходящий пресет настроек строгости;
- не заставлять нового пользователя сразу вручную настраивать все правила и флаги.

Первый шаг мастера:

`Выберите тип склада:`

- маленький прокат;
- растущая команда;
- большой склад.

После выбора система применяет стартовый пресет:
- `small_rental` для маленького проката;
- `growing_team` для растущей команды;
- `large_warehouse` для большого склада.

Правило:
- мастер первой настройки должен быть коротким и понятным;
- достаточно выбрать стартовый пресет и дать возможность позже изменить настройки;
- администратор или директор может после мастера вручную изменить правила по отдельным процессам;
- выбранный пресет не должен навсегда ограничивать ручные настройки.

Архитектурный смысл:
- склад проще запустить с нуля;
- маленькие команды быстрее получают рабочую конфигурацию;
- большие компании могут начать с более строгого набора правил;
- настройки остаются гибкими и масштабируемыми.

MVP-подход:
- показывать мастер при первой настройке склада в workspace;
- хранить факт прохождения мастера;
- применённый пресет сохранять как стартовую настройку правил процессов;
- дать администратору возможность вернуться в настройки и изменить правила вручную.
