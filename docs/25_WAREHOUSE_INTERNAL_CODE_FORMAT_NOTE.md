# 25.x. Формат внутреннего кода складской позиции

Этот документ дополняет `docs/25_WAREHOUSE_AUTO_INTERNAL_CODE_NOTE.md` и фиксирует решение 10.64.

## Решение

Основной внутренний код складской позиции должен генерироваться автоматически по схеме:

```text
<category-prefix>-<sequence>
```

Например:

```text
TRS-000123
SND-000245
LED-000044
PWR-000152
CBL-000333
```

## Логика

Код должен быть привязан к категории через короткий категорийный префикс.

Порядковый номер делает код уникальным внутри компании.

Код не должен пытаться хранить всю информацию о позиции.

Подробности должны оставаться в отдельных полях:

- название;
- категория;
- бренд;
- модель;
- артикул производителя;
- синонимы;
- комментарий.

## Почему так

Такой формат:

- короткий;
- стабильный;
- понятный человеку;
- удобный для QR / штрихкода;
- не ломается при переименовании позиции;
- не требует менять код при уточнении бренда, модели или описания.

## Настройки

Категорийный префикс задаётся в настройках категории.

Если категория меняется, внутренний код уже созданной позиции не должен автоматически меняться без отдельного осознанного действия.

## MVP

Для MVP достаточно автоматической генерации вида:

```text
CAT-000001
```

где `CAT` — префикс категории, а номер уникален внутри компании или внутри категории.
