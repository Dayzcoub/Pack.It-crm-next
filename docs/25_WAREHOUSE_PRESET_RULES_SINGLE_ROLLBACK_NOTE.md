# 25 Warehouse — Preset Rules Single Rollback Note

## 10.117. Ограничение отката настроек после применения пресета

Решение: откат настроек склада после применения пресета нужно ограничить одним шагом назад.

Правило:
- можно вернуть только состояние, которое было до последнего применённого пресета;
- многошаговую историю откатов на старте не делаем;
- после выполнения отката возможность повторного отката для этого же применения пресета считается использованной;
- если позже администратор снова применит новый пресет, система снова сохраняет новый snapshot для одного возможного отката.

Архитектурный смысл:
- откат нужен как страховка от неудачного применения пресета;
- сложная система версионирования настроек склада на MVP не нужна;
- один шаг назад покрывает основной практический сценарий и не усложняет интерфейс.

Что хранить:
- snapshot правил до последнего применения пресета;
- snapshot правил после применения пресета;
- статус: доступен ли откат;
- кто применил пресет;
- кто выполнил откат;
- дата и время действий.

MVP-подход:
- поддержать только `rollback_to_previous_preset_state`;
- не делать цепочку версий на много шагов назад;
- сохранять событие отката в истории настроек склада.
