# 25 Warehouse — Inventory Schedule Cancel Note

## 10.102. Отмена плановой инвентаризации

Решение: директору и администратору нужно разрешить полностью отменять плановую инвентаризацию.

Пример сценария:
- плановая проверка была создана по ошибке;
- проверка потеряла смысл из-за изменения складского процесса;
- область проверки уже была закрыта другой актуальной инвентаризацией;
- директор или администратор отменяет проверку вместо переноса.

Правило:
- отмена доступна только пользователям с повышенными правами: директор / администратор;
- обычный пользователь или профильный специалист не отменяет обязательную плановую проверку без соответствующих прав;
- отмена не равна завершению: задача не должна закрываться как выполненная;
- отменённая проверка должна сохраняться в истории, а не удаляться бесследно.

Что нужно хранить при отмене:
- кто отменил;
- когда отменил;
- исходная дата проверки;
- шаблон / область проверки;
- ответственный;
- комментарий причины отмены, если он будет включён отдельным правилом.

Архитектурный смысл:
- плановая инвентаризация может быть ошибочной или устаревшей;
- система должна позволять управленчески отменить такую проверку без имитации выполнения;
- история отмены нужна для прозрачности складского контроля и разбора спорных ситуаций.

MVP-подход:
- добавить статус плановой инвентаризации `отменено`;
- связанная календарная задача получает статус `отменено`;
- отменённая задача остаётся видимой в истории/архиве, но не требует выполнения и не считается просроченной.
