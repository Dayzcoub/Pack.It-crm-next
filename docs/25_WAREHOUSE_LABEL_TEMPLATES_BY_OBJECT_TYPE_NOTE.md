# 25.x. Шаблоны наклеек по типам объектов

Этот документ дополняет `docs/25_WAREHOUSE_LABEL_PRINT_SETTINGS_NOTE.md` и фиксирует решение 10.68.

## Решение

Для QR / штрихкод-наклеек нужны отдельные шаблоны под разные типы объектов.

Обычная складская позиция, конкретный экземпляр, кейс / контейнер и место хранения требуют разного набора данных на наклейке.

## Типы шаблонов

В архитектуре нужно заложить шаблоны для:

- общей складской позиции;
- конкретного экземпляра;
- кейса / контейнера;
- стеллажа / полки / места хранения.

## Примеры содержимого

Наклейка общей позиции может содержать:

- внутренний код позиции;
- название позиции;
- QR или штрихкод.

Наклейка экземпляра может содержать:

- внутренний код позиции;
- номер экземпляра;
- серийный номер, если есть;
- QR или штрихкод.

Наклейка кейса / контейнера может содержать:

- код контейнера;
- название контейнера;
- краткое содержимое;
- QR или штрихкод.

Наклейка места хранения может содержать:

- название локации;
- зона / стеллаж / полка / ячейка;
- QR или штрихкод места.

## Настройка

Администратор должен иметь возможность выбрать шаблон и поля, которые печатаются на наклейке.

## MVP

Для MVP достаточно базовых шаблонов для основных типов объектов и настройки отображаемых полей.

Сложный дизайнер наклеек можно не делать на старте.
