# 25 Warehouse — Inventory Schedule Reschedule Calendar History Note

## 10.101. История переносов плановой инвентаризации в календаре

Решение: переносы плановой инвентаризации нужно показывать в карточке связанной календарной задачи как отдельную историю изменений.

Пример отображения:
- было: 15 июля;
- перенесено на: 20 июля;
- причина: склад занят;
- кто перенёс: директор / администратор / конкретный пользователь;
- когда перенёс: дата и время изменения.

Правило:
- история переносов должна быть доступна прямо из календарной задачи плановой инвентаризации;
- системный журнал/audit log при этом тоже может хранить событие, но пользователь не должен идти только в технический журнал, чтобы понять, почему дата проверки изменилась;
- если переносов было несколько, карточка задачи должна показывать цепочку переносов в хронологическом порядке.

Архитектурный смысл:
- календарь остаётся понятной рабочей точкой для складского контроля;
- ответственный видит не только актуальную дату, но и историю управленческих изменений;
- это помогает разбирать просрочки, частые переносы и спорные ситуации без отдельного поиска в audit log.

MVP-подход:
- в карточке календарной задачи достаточно компактного блока истории переносов;
- для каждого переноса хранить старую дату, новую дату, пользователя, время изменения и комментарий причины;
- подробный технический audit log можно развивать отдельно, но календарная карточка должна давать базовое объяснение переноса.
