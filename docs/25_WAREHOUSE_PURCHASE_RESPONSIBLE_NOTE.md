# 25.x. Ответственный за закупку новой позиции

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.41.

## Решение

Для маленьких прокатов отдельный ответственный за закупку новой складской позиции не нужен.

На старте достаточно уже описанных данных покупки:

- дата покупки;
- стоимость покупки;
- поставщик / продавец;
- контакт;
- документы покупки;
- комментарий.

Отдельное поле `ответственный за закупку` не должно быть обязательной частью MVP.

Для больших компаний такая функция может быть полезна, но пока нет уверенности, что она нужна в базовом продукте.

Поэтому её можно заложить как возможное будущие расширение или опциональную настройку, но не усложнять стартовую складскую модель.
