# 25.x. Расходники и минимальные остатки

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.33.

## Решение

Для расходников на старте не делаем складской контроль минимального остатка основным сценарием.

Базовая схема расходников:

- расходники преимущественно покупаются под конкретный проект;
- расходники списываются на проект;
- если что-то осталось после проекта, дальнейшее использование остаётся на усмотрение работников.

Поэтому для MVP не нужно строить тяжёлую систему предупреждений вида `минимальный остаток / низкое количество` для расходников.

Если позже появится необходимость вести склад расходников более строго, можно добавить опциональные минимальные остатки и предупреждения в будущих версиях.
