# Документация PACK.IT CRM Next

Эта папка является основным местом хранения продуктовой, архитектурной и технической документации PACK.IT.

## Зачем нужна эта документация

Новая версия PACK.IT должна разрабатываться не как набор случайно нарастающих блоков, а как заранее описанная система:

- с понятной главной сущностью;
- с разделением продукта на модули;
- с нормальными ролями и доступами;
- с отдельной системной админкой;
- с подключаемым 3D-конструктором;
- с понятным складским контуром;
- с контролируемой генерацией документов;
- с возможностью развивать приложение без архитектурного хаоса.

## Главная сущность

Главная сущность приложения — **Проект / Смета**.

3D-конструктор, склад, документы, команда, транспорт, клиенты и статусы существуют вокруг проекта, а не заменяют его.

## Структура документации

- [`00_PRODUCT_PRINCIPLES.md`](00_PRODUCT_PRINCIPLES.md) — что такое PACK.IT и какие решения считаются базовыми.
- [`01_SYSTEM_ARCHITECTURE.md`](01_SYSTEM_ARCHITECTURE.md) — верхнеуровневая модульная архитектура приложения.
- [`02_PROJECT_QUOTE_DOMAIN.md`](02_PROJECT_QUOTE_DOMAIN.md) — доменная модель проекта/сметы.
- [`03_ADMIN_ROLES_ACCESS.md`](03_ADMIN_ROLES_ACCESS.md) — системная админка, роли, доступы и ключи.
- [`04_DOCUMENTATION_ROADMAP.md`](04_DOCUMENTATION_ROADMAP.md) — какие документы нужно подготовить дальше.

## Правило ведения документации

Все важные решения по PACK.IT должны переноситься из чата в документы.

Если в обсуждении появляется новое архитектурное решение, оно должно быть зафиксировано в одном из документов или вынесено в отдельный новый документ.
