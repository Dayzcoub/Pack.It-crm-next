# 17. Состав сметы

## Главное решение

После создания проекта пользователь попадает на экран состава сметы.

Сначала показываются выбранные разделы проекта. Состав можно редактировать: добавлять разделы, убирать разделы и возвращать их при изменении проекта.

## Базовые разделы

Базовые разделы:

- звук;
- свет;
- backline.

## Подключаемые разделы

По необходимости подключаются:

- сцена;
- фермы;
- LED;
- коммутация;
- питание 220/380;
- другое наполнение;
- услуги;
- расходники;
- транспорт;
- команда.

Сцена, фермы и LED могут подключаться через 3D-конструктор, если проект требует визуализации или технической сборки.

## Коммутация

В смете коммутация должна отображаться внутри соответствующего технического раздела.

Примеры:

- звук → коммутация звука;
- свет → коммутация света;
- LED → коммутация LED;
- сцена / фермы → связанная техническая коммутация, если она нужна.

Так смета становится нагляднее: видно, к какому разделу относится кабель, переходник или другая коммутационная позиция.

При этом в базе / каталоге категория должна оставаться одной общей категорией:

```text
Коммутация
```

Каталог не должен дублировать отдельные категории `коммутация звука`, `коммутация света`, `коммутация LED` как независимые справочники.

Раздел в смете определяется назначением позиции в конкретном проекте.

## Питание 220/380

Питание и силовая часть должны быть отдельной категорией, а не частью обычной коммутации.

К этой категории относятся:

- кабели 220 В;
- кабели 380 В;
- удлинители;
- распределители;
- силовые линии;
- щиты;
- силовые переходники;
- другая силовая инфраструктура.

Причина: у питания другая ответственность, требования безопасности, расчёт мощности и логика проверки нагрузки.

В базе это должна быть отдельная категория:

```text
Питание 220/380
```

В смете питание нужно показывать одной общей строкой или одним общим блоком, например:

```text
Силовая коммутация / питание 220/380 — вес X кг — цена Y
```

Питание не нужно раскидывать по разделам `Звук`, `Свет`, `LED`, если для клиента и менеджера достаточно общей силовой строки.

Детализация состава силовой части может быть доступна внутри технической расшифровки, склада или внутреннего состава, но клиентская смета должна оставаться компактной.

## Другое наполнение

Раздел `другое наполнение` нужно разбивать на подкатегории.

Он не должен быть общей кучей для всего подряд.

Стартовые подкатегории:

- мебель;
- шатры;
- декор;
- площадочные элементы;
- конструкции;
- прочее оборудование;
- прочее.

Подкатегории нужны, чтобы смета, КП, склад и аналитика оставались читаемыми.

## Ручные позиции

Ручные позиции должны быть отдельным разделом сметы.

Этот раздел нужен для позиций, которые не подходят ни под одну категорию компании, но должны быть указаны в смете и документах.

Ручная позиция не обязана автоматически становиться складской карточкой.

Из ручной позиции не нужно делать кнопку быстрого сохранения в каталог.

Если позиция часто повторяется, её нужно добавлять в базу / каталог напрямую, с заполнением всех обязательных параметров: веса, цены, мощности, категории, складской логики и других формальностей.
