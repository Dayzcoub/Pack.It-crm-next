# 25 Warehouse — Picking List Internal Only Note

## 10.135. Назначение сборочного листа

Решение: сборочный лист является только внутренним документом команды.

Правило:
- сборочный лист не предназначен для клиента;
- клиентскую версию сборочного листа не делаем;
- цены в сборочном листе не показываем;
- документ используется складом, техниками, менеджером проекта и другими внутренними участниками по правам.

Что может быть в сборочном листе:
- проект / мероприятие;
- дата и период выдачи;
- склад / локация выдачи;
- позиции и количества;
- кейсы / контейнеры;
- отметки собрано / не собрано;
- проблемные строки;
- комментарии для команды;
- ответственные.

Что не включать:
- клиентские цены;
- закупочные цены;
- коммерческие условия;
- публичные формулировки для клиента.

Архитектурный смысл:
- сборочный лист — рабочий инструмент подготовки оборудования;
- он должен быть удобным для команды, а не красивым коммерческим документом;
- отсутствие цен снижает риск случайного раскрытия внутренней экономики.

MVP-подход:
- сделать сборочный лист внутренним PDF/print документом;
- не делать отдельную клиентскую версию;
- не выводить цены ни в одной версии сборочного листа.
