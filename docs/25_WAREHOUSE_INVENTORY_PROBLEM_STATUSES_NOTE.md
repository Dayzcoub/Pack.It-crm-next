# 25.x. Статусы проблемных строк инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_MISSING_ITEM_PROBLEM_ROW_NOTE.md` и фиксирует решение 10.86.

## Решение

У проблемных строк инвентаризации должны быть статусы.

Статусы нужны, чтобы команда понимала, какие проблемы уже разобраны, какие находятся в работе, а какие требуют решения ответственного или руководителя.

## Базовые статусы

Для MVP можно заложить следующие статусы:

- новая;
- в работе;
- найдено / исправлено;
- не подтверждено;
- требует решения ответственного;
- эскалация администратору / директору;
- закрыто.

## Логика

Проблемная строка создаётся со статусом `новая`.

Назначенный ответственный переводит её в нужный статус по результатам разбора.

Если проблема решена, строка закрывается.

Если проблема требует списания, утери, спорного решения или финансовых последствий, она может быть переведена в статус эскалации.

## Права

Обычный профильный сотрудник может разбирать и закрывать простые проблемы в своей области, если это разрешено правами.

Критичные статусы и финальные решения по списанию, утере или спорным ситуациям требуют участия администратора или директора.

## MVP

Для MVP достаточно хранить:

- статус;
- ответственного;
- дату последнего изменения;
- комментарий;
- историю изменения статусов.
