# 25 Warehouse — Shortage List Compact Status Note

## 10.268. Compact status display in warehouse shortage list

Decision: shortage statuses should be visible in the warehouse keeper open shortage list, but must not clutter the interface.

Rules:
- status display in list should be compact;
- avoid large colorful badges that overload the list;
- use short text, small neutral badge, subtle icon, or restrained visual marker;
- detailed status and full history remain inside the shortage detail card;
- list should stay focused on project, missing item, quantity, and responsible/action state;
- mobile readability has priority.

Purpose:
- keep the interface clean;
- still make current state understandable at a glance;
- avoid visual noise in daily warehouse workflow.

MVP approach:
- show a compact status marker in each shortage row;
- use restrained styling, not aggressive color blocks;
- provide full status context after opening the detail card.
