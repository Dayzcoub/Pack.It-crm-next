# 25 Warehouse — Picking List Auto Category Split Note

## 10.137. Автоматическое разделение сборочного листа по категориям

Решение: сборочный лист по категориям должен строиться автоматически исходя из списка оборудования в проекте.

Правило:
- пользователь не должен вручную раскладывать позиции по отдельным сборочным листам;
- система берёт состав проекта и группирует позиции по складским категориям / направлениям;
- если в проекте есть звук, свет, LED, сцена, фермы или другие категории, по ним можно сформировать отдельные листы;
- категории берутся из карточек складских позиций и структуры проекта.

Назначение:
- уменьшить ручную работу менеджера и техника;
- исключить ошибки при ручном переносе позиций между листами;
- быстро отдавать профильным специалистам их часть проекта.

MVP-подход:
- общий сборочный лист проекта формируется из всего списка проекта;
- отдельные сборочные листы формируются автоматической группировкой по категории;
- ручного конструктора состава листа на старте не делать;
- цены не выводить, документ остаётся внутренним.
