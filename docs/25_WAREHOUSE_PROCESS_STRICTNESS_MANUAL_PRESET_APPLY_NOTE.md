# 25 Warehouse — Process Strictness Manual Preset Apply Note

## 10.113. Ручное применение пресета настроек позже

Решение: администратору нужно разрешить вручную выбрать и применить пресет настроек строгости склада позже.

Повторный мастер первой настройки склада не нужен. Вместо этого в административных настройках склада должно быть отдельное действие для выбора пресета.

Правила:
- действие доступно только администратору или директору;
- перед сохранением изменений нужно явное подтверждение;
- система должна сообщить, что правила складских процессов будут обновлены согласно выбранному пресету;
- после применения пресета отдельные правила процессов остаются доступными для ручного редактирования;
- если это не усложняет MVP, полезно показать список процессов, настройки которых будут обновлены.

Архитектурный смысл:
- компания может быстро перейти на другой стартовый набор правил без повторного мастера;
- пресет остаётся вспомогательным инструментом настройки;
- ручная настройка по отдельным процессам остаётся основной моделью дальнейшего развития склада.

История:
- сохранять, кто применил пресет;
- сохранять дату и время;
- сохранять название выбранного пресета;
- итоговые правила хранить как обычные редактируемые настройки процессов.

MVP-подход:
- добавить действие выбора пресета в административные настройки склада;
- требовать подтверждение перед сохранением;
- не запускать повторно мастер первой настройки.
