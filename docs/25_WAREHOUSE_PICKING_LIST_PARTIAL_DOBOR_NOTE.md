# 25 Warehouse — Picking List Partial Dobor Note

## 10.141. Недостающие позиции при частичной сборке

Решение: если позиция частично не собрана, не нужно сразу создавать проблемную строку. Достаточно комментария в сборочном листе и возможности вынести недостающее количество в добор на следующий заезд.

Правило:
- при частичной сборке пользователь указывает фактически собранное количество;
- недостающее количество остаётся видимым как остаток к добору;
- пользователь может оставить комментарий в сборочном листе;
- проблемная строка создаётся только если ситуация действительно требует разбора;
- для обычного недобора на момент сборки достаточно статуса и комментария.

Сценарий:
- нужно 10 кабелей;
- собрано 7;
- 3 кабеля будут добраны следующим заездом;
- в сборочном листе фиксируется комментарий и остаток к добору.

Назначение:
- не превращать каждую частичную сборку в проблему;
- поддержать реальные складские ситуации, когда оборудование довозится позже;
- сохранить понятную связь между текущей сборкой и следующим заездом.

MVP-подход:
- хранить picked quantity и remaining quantity;
- разрешить comment для частичной сборки;
- добавить возможность отметить remaining quantity as next trip / добор;
- не создавать problem row автоматически при каждой частичной сборке.
