# 25.x. Объект не найден при инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_AUDIT_NOTE.md` и фиксирует решение 10.84.

## Решение

Если объект при инвентаризации не найден, система должна создать проблемную строку для дальнейшего разбора.

Не нужно сразу автоматически списывать объект или переводить его в статус утери.

## Логика

При завершении проверки или при отметке `не найдено` система создаёт проблемную строку инвентаризации.

В проблемной строке нужно сохранить:

- объект;
- ожидаемое место хранения;
- область инвентаризации;
- кто проводил проверку;
- дату и время;
- комментарий, если есть;
- фото или файл, если есть.

## Дальнейший разбор

После появления проблемной строки ответственный пользователь может:

- найти объект позже и закрыть проблему;
- уточнить место хранения;
- перевести объект в ремонт или особый статус, если есть основания;
- создать запрос на списание или утерю, если объект действительно не найден после проверки.

## Почему так

Отсутствие объекта при инвентаризации не всегда означает утерю.

Он может быть в другой зоне, в кейсе, на проекте, у специалиста, на ремонте или неправильно привязан к месту хранения.

Поэтому сначала нужна проблемная строка для разбора, а не автоматическое списание.

## MVP

Для MVP достаточно:

1. отметить объект как `не найден`;
2. создать проблемную строку;
3. показать её в итогах инвентаризации;
4. дать ответственному пользователю обработать проблему позже.
