# 25 Warehouse — Picking List Interactive And Print Note

## 10.139. Отметки сборки в сборочном листе

Решение: сборочный лист должен поддерживать оба сценария — интерактивные отметки в интерфейсе и PDF/печать с пустыми клетками.

Правило:
- в интерфейсе сборочного листа должны быть чекбоксы / статусы `собрано` и `не собрано`;
- PDF/печатная версия должна иметь пустые клетки для ручных отметок;
- нельзя рассчитывать, что на каждом складе есть принтер;
- нельзя рассчитывать, что каждый склад всегда будет работать только с экраном.

Сценарии:
- склад без принтера работает с интерактивным листом на телефоне, планшете или ноутбуке;
- склад с бумажным процессом печатает PDF и отмечает позиции вручную;
- при необходимости команда может совмещать оба подхода.

Что фиксировать в интерфейсе:
- позиция собрана;
- позиция не собрана;
- частично собрана;
- комментарий / проблема, если что-то не сходится.

MVP-подход:
- сделать интерактивный сборочный лист в интерфейсе;
- сделать PDF/print версию с пустыми чекбоксами;
- PDF не содержит цен;
- сборочный лист остаётся внутренним документом.
