# 25 Warehouse — Transfer Delivered Notification Note

## 10.150. Notification when transfer has been delivered

Decision: when transfer equipment is marked as delivered, the receiving project team and project manager should be notified.

Rules:
- the notification is sent when the transfer status changes to delivered;
- the receiving project team should see that the equipment is already on site;
- the project manager should also receive the notification;
- the notification should reference the equipment, source project/site, target project/site, and responsible person;
- the transfer block remains the main place to see current transfer status.

Purpose:
- confirm that the transfer chain is closed;
- help the receiving team continue work without guessing whether equipment has arrived;
- keep the manager informed without requiring manual calls or messages.

MVP approach:
- send delivered notification to the target project manager and assigned target project team;
- store notification time and status change history;
- show delivered status in the transfer block of both related picking lists.
