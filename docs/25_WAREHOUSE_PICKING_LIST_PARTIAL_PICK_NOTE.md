# 25 Warehouse — Picking List Partial Pick Note

## 10.140. Частичная сборка позиции в интерактивном сборочном листе

Решение: интерактивный сборочный лист должен поддерживать частичную сборку позиции.

Правило:
- позиция может быть отмечена как собранная полностью;
- позиция может быть отмечена как не собранная;
- позиция может быть отмечена как частично собранная;
- для частичной сборки нужно указать фактически собранное количество;
- система должна видеть разницу между требуемым и собранным количеством.

Пример:
- требуется 10 кабелей;
- собрано 7;
- 3 не найдены или ждут добора.

Зачем это нужно:
- на складе может не хватать части позиции;
- часть оборудования может быть временно в другом месте;
- позицию могут дособрать позже;
- сборка больших проектов часто идёт в несколько проходов.

MVP-подход:
- хранить required quantity;
- хранить picked quantity;
- вычислять remaining quantity;
- показывать статус: not picked / partial / picked;
- разрешить комментарий или проблемную строку для недостающего количества;
- PDF/печать остаётся с пустыми клетками для ручной отметки, а интерактивный режим хранит точное количество.
