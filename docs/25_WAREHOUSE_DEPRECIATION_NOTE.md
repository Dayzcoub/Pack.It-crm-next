# 25.x. Амортизация оборудования

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.36.

## Решение

Для MVP не считаем амортизацию оборудования.

В карточке складской позиции или экземпляра достаточно хранить:

- дату покупки;
- стоимость покупки.

Поля срока службы, текущей остаточной стоимости и автоматического расчёта амортизации на старте не нужны.

Такой подход сохраняет полезную историю закупки, но не усложняет складской учёт бухгалтерской логикой.

Если в будущих версиях понадобится финансовая аналитика парка оборудования, расчёт амортизации можно добавить отдельным модулем или опциональной настройкой.
