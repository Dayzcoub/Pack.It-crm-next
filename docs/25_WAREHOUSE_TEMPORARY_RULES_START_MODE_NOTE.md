# 25 Warehouse — Temporary Rules Start Mode Note

## 10.130. Период запуска временного правила склада

Решение: временное правило склада можно включать как на будущий период, так и с текущего момента до выбранной даты окончания.

Поддерживаемые варианты:
- будущий период: `с 1 сентября по 30 сентября`;
- немедленный старт: `сейчас по 30 сентября`.

Правило:
- администратор выбирает дату и время начала;
- если временное правило нужно включить сразу, стартом становится текущий момент;
- дата и время окончания обязательны;
- временное правило можно отключить вручную до завершения периода;
- после завершения периода правило возвращается к исходному значению.

Архитектурный смысл:
- будущий период нужен для планируемых сезонных или проектных режимов;
- немедленный старт нужен для оперативного усиления процесса;
- оба сценария должны работать через одну модель временного правила.

MVP-подход:
- хранить `starts_at` и `ends_at`;
- поддержать быстрый вариант `start now`;
- не делать отдельные типы временных правил для будущего и текущего запуска.
