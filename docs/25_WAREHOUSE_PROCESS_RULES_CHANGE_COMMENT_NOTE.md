# 25 Warehouse — Process Rules Change Comment Note

## 10.121. Комментарий при ручном изменении правил склада

Решение: при ручном изменении правил склада администратор должен оставлять комментарий.

Сценарий:
- администратор меняет правило складского процесса;
- например включает более строгие требования для списаний, возвратов, проблемных строк или инвентаризации;
- система просит указать короткую причину изменения.

Что хранить:
- пользователь;
- дата и время;
- процесс;
- изменённое правило;
- старое значение;
- новое значение;
- комментарий администратора.

Правило:
- комментарий обязателен для ручных изменений правил склада;
- комментарий нужен не для обычных складских операций, а именно для изменения административных правил;
- история изменений доступна директору и администратору;
- обычный рабочий интерфейс склада не перегружается этими комментариями.

Архитектурный смысл:
- правила склада могут влиять на работу всей команды;
- позже должно быть понятно, почему правило было изменено;
- это особенно важно при переходе от гибкого процесса к более строгому.

MVP-подход:
- добавить обязательное поле comment/reason при сохранении ручного изменения правила;
- сохранять комментарий вместе с diff изменения;
- показывать комментарий только в административной истории настроек.
