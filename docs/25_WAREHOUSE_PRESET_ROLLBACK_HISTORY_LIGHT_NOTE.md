# 25 Warehouse — Preset Rollback Light History Note

## 10.119. История отката настроек пресета

Решение: отдельную подробную запись истории для отката настроек склада после применения пресета делать не нужно.

Правило:
- не создавать отдельный полноценный history item для отката;
- достаточно обновить запись события применения пресета;
- в этой записи можно отметить, что откат был использован;
- сохранить минимум: кто выполнил откат и когда;
- подробный список возвращённых правил отдельно не нужен.

Причина:
- откат доступен только один раз и только для последнего применённого пресета;
- отдельная сложная история отката на MVP избыточна;
- главное — понимать, что откат был использован, а не вести отдельную цепочку версий.

MVP-подход:
- у события применения пресета хранить поля `rollback_used`, `rollback_by`, `rollback_at`;
- не создавать отдельную ленту откатов;
- не выводить эту информацию в рабочий складской интерфейс.
