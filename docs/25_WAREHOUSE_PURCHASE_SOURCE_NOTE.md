# 25.x. Поставщик, продавец и документы покупки

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.37.

## Решение

Для складских позиций и экземпляров нужно хранить информацию о поставщике, продавце или контакте по покупке, если такая информация есть.

Эти данные нужны для истории происхождения оборудования, гарантии, сервиса, повторной закупки и возможных вопросов по документам.

На старте достаточно полей:

- поставщик / продавец / у кого купили;
- контактное лицо;
- телефон;
- документы покупки: чек / счёт / договор / файл;
- комментарий.

Поля не должны быть обязательными для каждой позиции, потому что часть оборудования может быть куплена давно, с рук или без полного комплекта документов.

Если информация есть, её нужно сохранять в карточке позиции или экземпляра.
