# 25.x. Уведомления по проблемным строкам инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_PROBLEM_DEADLINE_NOTE.md` и фиксирует решение 10.88.

## Решение

Уведомления по проблемным строкам инвентаризации нужно заложить в архитектуру, но не включать как обязательный механизм на старте.

Функция должна быть доступна позже, когда компании понадобится более строгий контроль разбора проблем.

## Возможные уведомления

В будущем система может уведомлять:

- назначенного ответственного о новой проблемной строке;
- ответственного о приближении дедлайна;
- ответственного о просрочке;
- администратора или директора о критичной просрочке;
- участников склада о смене статуса проблемной строки.

## Настройки

Уведомления должны быть настраиваемыми.

Компания может выбрать:

- не использовать уведомления;
- уведомлять только по критичным проблемам;
- уведомлять по всем новым проблемным строкам;
- уведомлять только при просрочке дедлайна.

## MVP

Для MVP уведомления не включаем как обязательную функцию.

Достаточно заложить поля и события, чтобы позже можно было подключить уведомления без переделки модели:

- ответственный;
- статус;
- дедлайн;
- дата изменения;
- событие создания проблемной строки;
- событие просрочки.
