# 25.x. Гарантия, сервис и мануалы

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.38.

## Решение

Для складских позиций и экземпляров нужно хранить гарантийную и сервисную информацию, если она есть или если позиция покупается новой.

На старте достаточно предусмотреть поля:

- гарантия до;
- сервисный интервал, если применимо;
- последнее обслуживание;
- комментарий по гарантии / сервису.

Эти поля не должны быть обязательными для старого оборудования или позиций, по которым такой информации нет.

Если позиция покупается новой, гарантийную информацию желательно заполнить сразу.

## Мануалы и официальные источники

Для складской позиции должна быть возможность прикрепить мануал или указать ссылку на официальный источник.

Варианты хранения:

- загрузить файл мануала;
- указать ссылку на страницу производителя или официальный источник;
- добавить комментарий к мануалу / источнику.

Мануалы нужны для технической подготовки, ремонта, проверки характеристик, питания, монтажа и безопасной эксплуатации оборудования.

Для MVP наличие мануала не обязательно, но возможность загрузки файла или ссылки должна быть заложена в архитектуру.
