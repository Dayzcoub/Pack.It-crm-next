# 25 Warehouse — First Run Setup Wizard No Rerun Note

## 10.112. Повторный запуск мастера настройки склада

Решение: повторно запускать мастер настройки склада позже не нужно.

Логика:
- мастер нужен только для первой настройки склада и выбора стартового пресета;
- по мере роста компании настройки будут меняться точечно;
- администратор должен менять отдельные правила под реальные условия, а не заново проходить общий сценарий настройки.

Причина:
- повторный мастер может случайно перетереть уже настроенные вручную правила;
- рост компании редко происходит одним резким переключением сценария;
- в реальности сначала меняется один процесс, потом другой: инвентаризация, возврат, списания, проблемные строки и т.д.;
- поэтому удобнее развивать настройки по отдельным процессам.

Правило:
- мастер первой настройки запускается один раз при первичной настройке склада;
- после прохождения мастер не нужен как повторяемый рабочий инструмент;
- все дальнейшие изменения выполняются через обычные настройки складских процессов;
- пресеты могут использоваться на старте, но не должны быть основным способом последующей перенастройки.

Архитектурный смысл:
- стартовый мастер ускоряет первичную настройку;
- дальнейшая эволюция склада должна идти через точечные правила;
- это снижает риск случайного сброса настроек и лучше соответствует реальному росту компании.

MVP-подход:
- хранить факт прохождения первого мастера;
- не делать кнопку `запустить мастер заново` как основную функцию;
- оставить ручные настройки по процессам для всех дальнейших изменений.
