# 25.x. Список последних сканов в мобильном сканере

Этот документ дополняет `docs/25_WAREHOUSE_MOBILE_SCANNER_MODE_NOTE.md` и фиксирует решение 10.76.

## Решение

В мобильном сканере не нужен отдельный список последних отсканированных объектов на старте.

Экран сканера должен оставаться простым и быстрым.

## Почему так

Список последних сканов может перегрузить мобильный интерфейс и отвлекать от основной задачи.

Для MVP важнее:

- быстро сканировать объект;
- получить понятное подтверждение результата;
- выполнить нужное действие;
- показать ошибку, если действие невозможно.

## Где смотреть историю

История действий должна сохраняться в соответствующей складской операции:

- сборочный лист;
- выдача;
- возврат;
- инвентаризация;
- карточка объекта;
- проблемная строка.

Отдельный список последних сканов в самом сканере не обязателен.

## MVP

Для MVP в мобильном сканере достаточно:

- сканирования;
- быстрого меню действий;
- визуального / звукового / вибро-подтверждения;
- сообщения об ошибке.

Если позже появится потребность, список последних сканов можно добавить как опциональную настройку.
