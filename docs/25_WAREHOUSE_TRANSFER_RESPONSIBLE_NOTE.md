# 25 Warehouse — Transfer Responsible Note

## 10.144. Ответственный за переезд оборудования между проектами

Решение: для переезда оборудования между проектами нужно указывать ответственного.

Сценарий:
- один комплект оборудования используется на двух проектах в один день;
- после первого проекта комплект должен быть забран с площадки A;
- затем комплект должен быть доставлен на площадку B;
- нужен понятный ответственный за этот переезд.

Правило:
- transfer-строка должна иметь ответственного пользователя или роль;
- ответственный может быть водитель, техник, менеджер, старший смены или другой назначенный участник;
- назначение ответственного помогает понять, кто отвечает за забор, перевозку и передачу оборудования;
- отсутствие ответственного может быть предупреждением, но не обязательно блокером для MVP.

Что хранить:
- source project / площадка отправления;
- target project / площадка назначения;
- equipment / позиция или комплект;
- planned pickup time;
- planned delivery time;
- responsible user / responsible role;
- transfer comment;
- transfer status.

MVP-подход:
- добавить поле responsible для transfer between projects;
- показывать ответственного в transfer-листе и связанных сборочных листах;
- не делать сложный workflow логистики на старте;
- достаточно статуса и понятного назначения ответственного.
