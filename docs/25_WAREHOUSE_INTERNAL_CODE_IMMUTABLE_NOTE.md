# 25.x. Изменение уже созданного внутреннего кода

Этот документ дополняет `docs/25_WAREHOUSE_INTERNAL_CODE_FORMAT_NOTE.md` и фиксирует решение 10.66.

## Решение

Ручное изменение уже созданного внутреннего кода складской позиции не разрешаем.

Внутренний код должен оставаться стабильным идентификатором позиции внутри компании.

## Почему так

Если код менять вручную после создания позиции, можно сломать или запутать:

- историю складских операций;
- старые проекты;
- сборочные листы;
- акты выдачи и возврата;
- связи с BOM и конструкторами;
- QR / штрихкоды;
- поиск и импорт/экспорт каталога.

## Правило

Код генерируется автоматически при создании позиции и далее не меняется.

Если пользователь хочет изменить название, бренд, модель, категорию, артикул производителя или синонимы — это можно делать в соответствующих полях, не меняя внутренний код.

## Исключение

Если код был создан ошибочно из-за системной ошибки или миграции, исправление возможно только через специальное административное или техническое действие, не как обычная функция интерфейса.

## MVP

В MVP внутренний код считается неизменяемым после создания позиции.
