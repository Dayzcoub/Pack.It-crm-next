# 18. Инфраструктура площадки

## Решение

`Инфраструктура площадки` — отдельный раздел сметы, а не подкатегория `другого наполнения`.

Этот раздел нужен для позиций, которые обеспечивают работу площадки, питание, тепло, безопасность, доступ и техническую инфраструктуру мероприятия.

## Стартовые подкатегории

На старте в раздел входят:

- генераторы;
- тепловые пушки;
- утяжелители / ФБС;
- ограждения;
- кабельные трапы;
- площадочные элементы;
- прочая инфраструктура.

## Почему это отдельный раздел

Генераторы и тепловые пушки нельзя оставлять просто в `прочем оборудовании`, потому что у них есть отдельные технические и операционные параметры:

- мощность;
- топливо / расход топлива;
- безопасность;
- доставка и погрузка;
- контроль запуска / работы;
- ответственность на площадке;
- связь с питанием 220/380 и общей инфраструктурой мероприятия.

## Отображение клиенту

В клиентской смете `Инфраструктура площадки` может показываться отдельным блоком, чтобы клиент видел, что это не звук, не свет, не сцена и не декор, а обеспечение площадки.

Пример:

```text
Инфраструктура площадки
Генератор — цена Y
Тепловая пушка — цена Y
Утяжелители / ФБС — цена Y
```

## Внутреннее использование

Внутри проекта позиции инфраструктуры используются для:

- технических листов;
- расчёта мощности;
- расчёта топлива, если применимо;
- склада и комплектации;
- логистики и транспорта;
- задач команды;
- контроля ответственности на площадке.

## Генераторы

Генератор как позиция инфраструктуры должен хранить не только коммерческую цену, но и технические параметры, которые нужны для питания, топлива, логистики и ответственности на площадке.

Стартовые поля карточки генератора:

- мощность, кВт;
- тип топлива;
- расход топлива;
- время работы;
- требуется оператор: да / нет.

Поле `мощность, кВт` используется внутри проекта для связи с расчётом питания 220/380 и проверки нагрузки.

Поля `тип топлива`, `расход топлива` и `время работы` нужны для внутреннего расчёта топлива, подготовки площадки и понимания эксплуатационных расходов.

Поле `требуется оператор` нужно для задач команды, ответственности и понимания, нужен ли отдельный специалист на площадке.

Клиенту эти технические параметры по умолчанию не раскрываются в смете. В клиентской смете генератор может отображаться простой строкой внутри раздела `Инфраструктура площадки`, если менеджер специально не решил раскрыть техническое приложение.

## Тепловые пушки

Тепловая пушка как позиция инфраструктуры должна хранить параметры, которые нужны для подбора тепла, расчёта питания или топлива, логистики и контроля безопасной работы на площадке.

Стартовые поля карточки тепловой пушки:

- тип пушки: электрическая / дизельная / газовая;
- тепловая мощность, кВт;
- потребляемая электрическая мощность, кВт;
- тип топлива;
- расход топлива;
- время работы;
- требуется оператор / контроль: да / нет.

Поле `тип пушки` определяет, какие дополнительные параметры важны для конкретной позиции: для электрической — питание и электрическая мощность, для дизельной или газовой — топливо, расход и контроль безопасной работы.

Поле `тепловая мощность, кВт` используется для понимания, какую задачу по обогреву закрывает позиция.

Поле `потребляемая электрическая мощность, кВт` используется внутри проекта для связи с расчётом питания 220/380 и проверкой нагрузки, особенно для электрических пушек и вспомогательных систем дизельных / газовых пушек.

Поля `тип топлива`, `расход топлива` и `время работы` нужны для внутреннего расчёта топлива, подготовки площадки и понимания эксплуатационных расходов.

Поле `требуется оператор / контроль` нужно для задач команды, ответственности и контроля безопасной работы оборудования на площадке.

Клиенту эти технические параметры по умолчанию не раскрываются в смете. В клиентской смете тепловая пушка может отображаться простой строкой внутри раздела `Инфраструктура площадки`, если менеджер специально не решил раскрыть техническое приложение.

## Утяжелители / ФБС / балласт

Утяжелители, ФБС и другой балласт как позиции инфраструктуры должны хранить параметры, которые нужны для расчёта веса, транспорта, спецпогрузки, манипулятора и понимания, какую техническую задачу они закрывают на площадке.

Стартовые поля карточки балласта:

- тип балласта: ФБС / бетонный блок / водоналивной / металлический / другое;
- вес одной единицы, кг;
- габариты;
- нужна спецпогрузка: да / нет;
- нужен манипулятор: да / нет;
- точка применения: сцена / фермы / шатёр / ограждения / другое;
- комментарий по размещению.

Поле `тип балласта` нужно для каталога, подбора и понятного отображения позиции внутри проекта.

Поле `вес одной единицы, кг` используется во внутренних расчётах веса, загрузки, транспорта и логистики. Клиенту вес по умолчанию не показывается.

Поле `габариты` нужно для планирования погрузки, размещения на площадке и понимания, поместится ли позиция в выбранный транспорт.

Поля `нужна спецпогрузка` и `нужен манипулятор` нужны для автоматических подсказок по транспорту, услугам, задачам и внутренней себестоимости проекта.

Поле `точка применения` связывает балласт с конкретной задачей проекта: сценой, фермами, шатром, ограждениями или другой площадочной конструкцией.

Поле `комментарий по размещению` нужно для технических листов, плана площадки и указаний команде.

Клиенту эти технические параметры по умолчанию не раскрываются в смете. В клиентской смете балласт может отображаться простой строкой внутри раздела `Инфраструктура площадки`, если менеджер специально не решил раскрыть техническое приложение.

## Ограждения

Ограждения как позиции инфраструктуры должны хранить параметры, которые нужны для расчёта количества секций, логистики, расстановки, веса, назначения зон и подготовки технического плана площадки.

Стартовые поля карточки ограждения:

- тип ограждения: барьер безопасности / мобильное ограждение / сигнальное / декоративное / другое;
- длина одной секции, м;
- высота, м;
- вес одной секции, кг;
- способ установки: свободностоящее / на базе / с креплением / другое;
- назначение: зрительская зона / backstage / периметр / техника / другое;
- комментарий по расстановке.

Поле `тип ограждения` нужно для каталога, подбора и понятного отображения позиции внутри проекта.

Поле `длина одной секции, м` используется для расчёта количества секций по нужной длине периметра или зоны.

Поле `высота, м` нужно для понимания назначения ограждения и его применимости на площадке.

Поле `вес одной секции, кг` используется во внутренних расчётах веса, загрузки, транспорта и логистики. Клиенту вес по умолчанию не показывается.

Поле `способ установки` нужно для задач команды, времени монтажа, крепежа, баз и проверки, можно ли использовать ограждение на конкретной площадке.

Поле `назначение` связывает ограждение с конкретной зоной проекта: зрительской зоной, backstage, периметром, технической зоной или другим назначением.

Поле `комментарий по расстановке` нужно для технических листов, плана площадки и указаний команде.

Клиенту эти технические параметры по умолчанию не раскрываются в смете. В клиентской смете ограждения могут отображаться простой строкой внутри раздела `Инфраструктура площадки`, если менеджер специально не решил раскрыть техническое приложение.

## Кабельные трапы

Кабельные трапы как позиции инфраструктуры должны хранить параметры для расчёта протяжённости трассы, количества секций, допустимого сценария использования, логистики и прокладки кабельных линий на площадке.

Стартовые поля карточки кабельного трапа:

- тип трапа: лёгкий / тяжёлый / резиновый / модульный / другое;
- длина одной секции, м;
- ширина, м;
- количество каналов;
- максимальная нагрузка / проезд: пешеходы / легковой транспорт / грузовой транспорт / другое;
- вес одной секции, кг;
- комментарий по прокладке.

Поле `длина одной секции, м` используется для расчёта количества секций по нужной длине кабельной трассы.

Поле `количество каналов` нужно для понимания, сколько кабельных линий можно развести через трап.

Поле `максимальная нагрузка / проезд` нужно для проверки сценария применения трапа на площадке.

Поле `вес одной секции, кг` используется во внутренних расчётах веса, загрузки, транспорта и логистики. Клиенту вес по умолчанию не показывается.

Поле `комментарий по прокладке` нужно для технических листов, плана площадки и указаний команде.

Клиенту эти технические параметры по умолчанию не раскрываются в смете. В клиентской смете кабельные трапы могут отображаться простой строкой внутри раздела `Инфраструктура площадки`, если менеджер специально не решил раскрыть техническое приложение.

## Площадочные элементы

Площадочные элементы — запасная подкатегория инфраструктуры для редких временных технических вещей площадки.

Стартовые поля карточки площадочного элемента:

- тип элемента: тент / навес / рабочий стол / стойка / временная зона / другое;
- габариты;
- вес, кг;
- назначение: FOH / backstage / склад на площадке / техзона / входная зона / другое;
- требуется сборка: да / нет;
- требуется крепёж: да / нет;
- комментарий по размещению.

Эта подкатегория может использоваться редко, но нужна как структурированный запас, чтобы редкие площадочные позиции не попадали в хаотичное `прочее`.

## Прочая инфраструктура

`Прочая инфраструктура` остаётся fallback-подкатегорией для редких инфраструктурных позиций, которые пока не имеют отдельной подкатегории и не требуют собственной технической карточки.

На старте для неё не нужны отдельные технические поля. Используется универсальная ручная карточка:

- название;
- количество;
- единица измерения;
- цена клиенту;
- внутренняя себестоимость;
- комментарий;
- показывать клиенту: да / нет.

Если позиция из `прочей инфраструктуры` начинает часто повторяться, её нужно перенести в нормальную подкатегорию или завести отдельную карточку каталога с полными параметрами.

## Связь с другими разделами

`Инфраструктура площадки` не заменяет раздел `Питание 220/380`.

Генератор как оборудование живёт в инфраструктуре площадки, а кабели, щиты, силовые линии и распределение питания живут в разделе `Питание 220/380`.

Связь между ними должна быть внутренней технической связью проекта, а не смешиванием категорий в клиентской смете.
