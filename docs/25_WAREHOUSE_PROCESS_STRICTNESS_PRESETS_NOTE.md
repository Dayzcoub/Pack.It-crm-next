# 25 Warehouse — Process Strictness Presets Note

## 10.110. Пресеты настроек строгости склада

Решение: для складского блока нужно сделать готовые пресеты настроек строгости.

Пресеты нужны как быстрый старт, но не как жёсткое ограничение. После выбора пресета администратор должен иметь возможность вручную подкрутить правила по отдельным процессам.

Базовые пресеты:

### Маленький прокат

Подходит для небольшой команды, где важны скорость и минимум лишней бюрократии.

Ориентировочная логика:
- большинство процессов в гибком режиме;
- минимум обязательных комментариев;
- переносы и отмены плановых проверок доступны директору/админу;
- история сохраняется, но интерфейс не перегружается деталями;
- строгие правила включены только для критичных действий: списания, утери, подтверждение проблемных ситуаций.

### Растущая команда

Подходит для компании, где уже появляются роли, ответственные и более регулярный складской процесс.

Ориентировочная логика:
- инвентаризация может быть строже, чем выдача и обычный возврат;
- комментарии требуются для переносов, отмен, проблемных строк и спорных операций;
- дедлайны и уведомления можно включать точечно;
- история изменений показывается там, где это помогает работе.

### Большой склад

Подходит для компаний с отдельными складскими работниками, несколькими зонами хранения и формальным контролем.

Ориентировочная логика:
- больше процессов работают в строгом режиме;
- обязательные комментарии для важных действий;
- можно включать лимиты переносов, эскалации, дедлайны и расширенную историю;
- проблемные строки, утери, списания и просрочки контролируются строже.

Правило:
- пресет задаёт стартовый набор правил;
- администратор может изменить отдельные настройки после применения пресета;
- пресет не должен скрывать ручную настройку;
- повторное применение пресета должно быть отдельным осознанным действием, чтобы случайно не затереть ручные настройки.

Архитектурный смысл:
- новый пользователь быстрее запускает складской блок;
- маленькие команды не получают избыточно строгую систему;
- большие компании могут быстро включить более формальный контур;
- настройка остаётся масштабируемой и понятной.

MVP-подход:
- заложить сущность/конфиг пресетов `warehouse_process_rule_presets`;
- предусмотреть пресеты `small_rental`, `growing_team`, `large_warehouse`;
- после применения пресета хранить итоговые правила по процессам как обычные настройки, доступные для ручного редактирования.
