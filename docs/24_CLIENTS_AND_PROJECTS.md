# 24. Клиенты и проекты

## Решение

`Клиенты и проекты` — базовый стартовый блок CRM.

Клиент и проект создаются до состава сметы, потому что все дальнейшие разделы сметы, логистики, команды, склада, документов и финансов привязываются к конкретному проекту.

## Клиент

Клиент нужен как основная сущность, к которой привязываются проекты, контакты, история, документы, счета, сметы и коммуникации.

На старте клиент может быть выбран из базы или создан в процессе создания проекта.

## Тип клиента

На старте у клиента есть тип:

- физическое лицо;
- юридическое лицо / организация.

Тип клиента нужен для будущей логики документов, реквизитов, договоров, счетов и отображения данных в проекте.

Для MVP тип клиента не должен усложнять создание проекта: достаточно простого выбора типа при создании или редактировании клиента.

## Физическое лицо

Для клиента типа `физическое лицо` на старте нужны поля:

- имя;
- фамилия;
- телефон;
- email;
- комментарий.

Этих полей достаточно для быстрого создания клиента, связи с проектом, коммуникаций и базового документооборота на старте.

## Юридическое лицо / организация

Для клиента типа `юридическое лицо / организация` карточку нужно разделить на чистую основную часть и отдельный блок реквизитов.

В основной карточке организации на старте нужны поля:

- название организации;
- контактное лицо;
- телефон;
- email;
- комментарий.

ИНН, КПП и остальные реквизиты юридического лица не должны засорять основную карточку.

Реквизиты нужно сгруппировать в отдельном окошке / блоке карточки клиента, чтобы интерфейс оставался чистым.

## Реквизиты юридического лица

Реквизиты юридического лица должны находиться в отдельном блоке / окошке карточки клиента.

На старте в блоке реквизитов нужны поля:

- ИНН;
- КПП;
- ОГРН;
- юридический адрес;
- фактический адрес;
- банковские реквизиты;
- комментарий по реквизитам.

Также нужна возможность загрузить файл с реквизитами.

Файл с реквизитами должен быть доступен для просмотра из карточки клиента.

Это позволит быстро прикреплять готовые реквизиты клиента, не засоряя основную карточку организации большим количеством полей.

## Проект

Проект — рабочая единица CRM.

К проекту привязываются:

- состав сметы;
- оборудование;
- услуги;
- команда;
- транспорт;
- субаренда;
- расходники;
- питание 220/380;
- инфраструктура площадки;
- площадка и зал;
- документы;
- задачи;
- финансы;
- комментарии;
- история изменений.

## Поля проекта при создании

Для MVP при создании проекта нужны поля:

- название проекта;
- клиент;
- дата мероприятия;
- площадка / адрес;
- зал / пространство площадки, если есть;
- ответственный менеджер;
- комментарий.

Этого достаточно, чтобы быстро создать проект, связать его с клиентом и перейти к составу сметы.

## Статусы проекта

На старте у проекта есть простой список статусов:

- черновик;
- в работе;
- согласован;
- выполнен;
- отменён.

Этого достаточно для MVP, чтобы понимать состояние проекта без сложного workflow.

## Площадки

Площадка должна быть отдельной сущностью / базой, а не только текстовым полем в проекте.

Разные клиенты могут проводить мероприятия на одной и той же площадке, поэтому площадку нужно выбирать из базы прошлых площадок или создавать новую.

У площадки могут быть разные залы / пространства для проведения мероприятий.

Пример:

```text
Площадка: Дом культуры
Зал: Большой зал
Зал: Малый зал
Зал: Фойе
```

В проекте выбирается площадка и, если нужно, конкретный зал / пространство.

Это позволит копить историю проектов по площадке, быстрее заполнять адреса и в будущем хранить технические особенности площадки.

## Карточка площадки

Для карточки площадки на старте нужны поля:

- название площадки;
- адрес;
- контакт площадки;
- телефон;
- комментарий;
- список залов / пространств.

Также в карточке площадки нужно хранить местных специалистов и полезные контакты.

Примеры локальных контактов:

- администратор площадки;
- электрик;
- технический директор площадки;
- ответственный за зал;
- охрана / пропускной режим;
- другой местный специалист.

Для каждого такого контакта желательно хранить:

- роль / должность;
- имя;
- телефон;
- комментарий.

Это нужно, чтобы менеджер или техдир быстро находил нужного человека на площадке при подготовке проекта и на монтаже.

## Создание проекта

Перед переходом к составу сметы нужно выбрать или создать клиента и создать проект.

После создания проекта пользователь попадает на экран состава сметы, где выбирает разделы проекта и наполняет их.

## Роль клиента в других разделах

Выбранный клиент используется в других разделах проекта.

Примеры:

- в самовывозе достаточно клиента, выбранного при создании проекта;
- в документах клиент подставляется в КП, договоры и счета;
- в истории клиента сохраняются прошлые проекты;
- в коммуникациях проектные сообщения можно связать с клиентом.

## MVP-ограничение

На старте не нужно превращать блок клиентов в тяжёлую enterprise CRM.

Главная задача MVP:

- быстро создать клиента;
- быстро создать проект;
- перейти к составу сметы;
- сохранить связь проекта со всеми дальнейшими разделами.
