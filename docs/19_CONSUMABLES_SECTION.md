# 19. Расходники

## Решение

`Расходники` — отдельный раздел сметы и внутренний рабочий список проекта.

Расходники не должны быть одной плоской кучей. Внутри проекта они ведутся подробным списком с подкатегориями, чтобы было понятно, что нужно купить, подготовить, списать, довезти или учесть в себестоимости.

В клиентской смете расходники по умолчанию выводятся одной общей строкой или одним общим блоком:

```text
Расходные материалы — цена Y
```

Клиенту не нужно по умолчанию раскрывать техническую детализацию расходников, если менеджер специально не решил показать подробную расшифровку.

## Стартовые подкатегории

На старте в раздел `Расходники` входят:

- топливо;
- крепёж;
- скотч / ленты / маркировка;
- кабельные стяжки;
- батарейки / аккумуляторы;
- упаковка / плёнка / стрейч;
- хозрасходники;
- прочее.

## Топливо

Топливо остаётся в расходниках, даже если используется для генераторов или тепловых пушек из раздела `Инфраструктура площадки`.

Генераторы и тепловые пушки должны иметь внутреннюю связь с расходниками по топливу, чтобы система могла считать потребность по:

- типу топлива;
- расходу;
- времени работы;
- запасу.

В клиентской смете топливо по умолчанию не раскрывается отдельной технической строкой. Оно попадает в общий блок расходников, если менеджер специально не решил раскрыть его клиенту отдельно.

## Крепёж

Крепёж в расходниках — это мелкий расходный крепёж для площадки, монтажа и временных задач, а не основной специализированный крепёж оборудования.

К расходному крепежу относятся:

- саморезы;
- дюбели;
- анкеры;
- болты / гайки / шайбы;
- хомуты;
- прочий монтажный крепёж.

Специализированный крепёж оборудования, который является частью комплектовки фермы, сцены, LED или другого технического раздела, должен считаться в соответствующем техническом разделе.

Примеры такого специализированного крепежа:

- C2 / CQ2 и связанный крепёж ферм;
- крепёж LED-конструкций;
- крепёж, который автоматически следует из сборки сцены или другого конструктора.

Такой крепёж не нужно вручную добавлять в расходники, если он уже рассчитывается техническим модулем.

## Скотч / ленты / маркировка

`Скотч / ленты / маркировка` — отдельная подкатегория расходников для монтажных, маркировочных и временных задач на площадке.

Стартовые типы:

- gaffer tape;
- изолента;
- сигнальная лента;
- маркировочная лента;
- двусторонний скотч;
- малярная лента;
- прочее.

Минимальные поля карточки:

- тип;
- цвет;
- длина / рулон;
- количество;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Эта подкатегория нужна для закупки, комплектации, маркировки кабелей, временной фиксации, обозначения зон и других монтажных задач.

Клиенту эти позиции по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Кабельные стяжки

`Кабельные стяжки` — отдельная простая подкатегория расходников для крепления, группировки и временной фиксации кабелей и мелких элементов на площадке.

Минимальные поля карточки:

- длина;
- ширина;
- цвет;
- материал / тип: обычная / усиленная / многоразовая / металлическая / другое;
- упаковка / количество в пачке;
- количество пачек;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Эта карточка нужна, чтобы не терять важные параметры стяжек: размер, цвет, тип и фасовку.

Клиенту кабельные стяжки по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Батарейки / аккумуляторы

`Батарейки / аккумуляторы` — отдельная подкатегория расходников для питания радиосистем, пультов, мелких устройств, подсветки, реквизита и других временных задач проекта.

Стартовые типы:

- AA;
- AAA;
- 9V / крона;
- CR2032;
- аккумулятор AA;
- аккумулятор AAA;
- прочее.

Минимальные поля карточки:

- тип;
- напряжение;
- ёмкость, если известно;
- количество в упаковке;
- количество упаковок;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Эта карточка нужна для закупки, комплектации и контроля внутренних расходов по автономному питанию.

Клиенту батарейки и аккумуляторы по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Упаковка / плёнка / стрейч

`Упаковка / плёнка / стрейч` — отдельная подкатегория расходников для упаковки, защиты, временного хранения и перевозки оборудования.

Стартовые типы:

- стрейч-плёнка;
- пузырчатая плёнка;
- картон / коробки;
- пакеты / мешки;
- защитная плёнка;
- прочее.

Минимальные поля карточки:

- тип;
- размер / формат;
- количество;
- единица измерения: рулон / коробка / пачка / метр / штука;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Эта карточка нужна для закупки, комплектации, защиты оборудования, упаковки кабелей, временного хранения и подготовки транспорта.

Клиенту упаковка и плёнки по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Хозрасходники

`Хозрасходники` — простая бытовая и организационная подкатегория расходников для обеспечения монтажных работ, уборки, мелкой подготовки и работы команды на площадке.

Стартовые типы:

- перчатки;
- салфетки / тряпки;
- мусорные пакеты;
- чистящие средства;
- вода техническая;
- бумага / ручки / канцелярия;
- прочее.

Минимальные поля карточки:

- тип;
- количество;
- единица измерения;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Клиенту хозрасходники по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Прочее

`Прочее` в расходниках остаётся fallback-подкатегорией для редких расходных позиций, которые пока не подходят под существующие подкатегории.

Для `прочего` не нужны отдельные специальные поля. Используется универсальная карточка:

- название;
- количество;
- единица измерения;
- цена клиенту;
- внутренняя себестоимость;
- комментарий;
- показывать клиенту: да / нет.

Если позиция из `прочего` начинает часто повторяться, её нужно перенести в нормальную подкатегорию или завести отдельную карточку каталога с полными параметрами.

## Попадание расходников в проект

Расходники могут попадать в проект двумя способами:

1. Ручное добавление менеджером или техником.
2. Автоматическая подсказка / расчёт из технических разделов проекта.

Стартовые источники расходников:

- `manual` — добавлено вручную;
- `generator_fuel` — топливо из генератора;
- `heater_fuel` — топливо из тепловой пушки;
- `cable_route` — расходники из кабельных трасс;
- `logistics_packaging` — упаковка из логистики / подготовки оборудования;
- `mounting_task` — расходники из монтажных задач.

Автоматически добавленные расходники должны хранить источник, чтобы было понятно, что добавил человек, а что рассчитала система.

Это стартовое правило для альфы. При тестировании системы источники, названия source-меток и правила автодобавления можно менять без переделки всей структуры сметы.

## Ручное изменение авторасходников

Ручное изменение автоматически добавленного расходника имеет приоритет над автоматическим расчётом.

Если пользователь вручную изменил авторасходник, позиция должна сохранить исходный источник и получить статус ручного изменения:

```text
source: generator_fuel
status: manual_override
```

После ручного изменения система не должна молча перетирать эту позицию следующим автоматическим пересчётом.

Нужно хранить аудит ручного изменения:

- кто изменил;
- когда изменил;
- что было изменено;
- комментарий, если пользователь его оставил.

При следующем пересчёте система может показать предупреждение, что автоматический расчёт отличается от ручного значения, но финальное значение остаётся ручным, пока пользователь сам не вернёт автоматический расчёт.

## Возврат авторасчёта

Для авторасходника со статусом `manual_override` нужна явная кнопка / действие:

```text
Вернуть авторасчёт
```

После этого система снова берёт значение из исходного автоматического источника, например генератора, тепловой пушки, кабельной трассы, логистики или монтажной задачи.

При возврате авторасчёта нужно:

- снять статус `manual_override`;
- пересчитать значение по исходному `source`;
- сохранить запись в аудите: кто вернул авторасчёт и когда;
- при необходимости сохранить историю старого ручного значения.

Это действие должно быть осознанным: система не должна сама возвращать авторасчёт без команды пользователя.

## Внутреннее использование

Подробный список расходников нужен для:

- закупки;
- склада;
- комплектации;
- списания;
- загрузки;
- технических листов;
- внутренней себестоимости проекта;
- контроля маржи.

## Отображение клиенту

По умолчанию клиент видит только общий блок расходников.

Логика:

```text
Подробный список расходников в проекте
→ внутренние технические, складские и закупочные листы
→ одна итоговая строка в клиентской смете
```

Если менеджер хочет раскрыть отдельный расходник клиенту, это должно быть ручным решением на уровне позиции или блока.

## Скидка

Расходники по умолчанию исключаются из базы скидки.

Причина: у расходников часто есть прямая закупочная или списываемая себестоимость, и автоматическая скидка может съедать маржу или создавать убыток.
