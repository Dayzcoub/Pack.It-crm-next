# 19. Расходники

## Решение

`Расходники` — отдельный раздел сметы и внутренний рабочий список проекта.

Расходники не должны быть одной плоской кучей. Внутри проекта они ведутся подробным списком с подкатегориями, чтобы было понятно, что нужно купить, подготовить, списать, довезти или учесть в себестоимости.

В клиентской смете расходники по умолчанию выводятся одной общей строкой или одним общим блоком:

```text
Расходные материалы — цена Y
```

Клиенту не нужно по умолчанию раскрывать техническую детализацию расходников, если менеджер специально не решил показать подробную расшифровку.

## Стартовые подкатегории

На старте в раздел `Расходники` входят:

- топливо;
- крепёж;
- скотч / ленты / маркировка;
- кабельные стяжки;
- батарейки / аккумуляторы;
- упаковка / плёнка / стрейч;
- хозрасходники;
- прочее.

## Топливо

Топливо остаётся в расходниках, даже если используется для генераторов или тепловых пушек из раздела `Инфраструктура площадки`.

Генераторы и тепловые пушки должны иметь внутреннюю связь с расходниками по топливу, чтобы система могла считать потребность по:

- типу топлива;
- расходу;
- времени работы;
- запасу.

В клиентской смете топливо по умолчанию не раскрывается отдельной технической строкой. Оно попадает в общий блок расходников, если менеджер специально не решил раскрыть его клиенту отдельно.

## Крепёж

Крепёж в расходниках — это мелкий расходный крепёж для площадки, монтажа и временных задач, а не основной специализированный крепёж оборудования.

К расходному крепежу относятся:

- саморезы;
- дюбели;
- анкеры;
- болты / гайки / шайбы;
- хомуты;
- прочий монтажный крепёж.

Специализированный крепёж оборудования, который является частью комплектовки фермы, сцены, LED или другого технического раздела, должен считаться в соответствующем техническом разделе.

Примеры такого специализированного крепежа:

- C2 / CQ2 и связанный крепёж ферм;
- крепёж LED-конструкций;
- крепёж, который автоматически следует из сборки сцены или другого конструктора.

Такой крепёж не нужно вручную добавлять в расходники, если он уже рассчитывается техническим модулем.

## Скотч / ленты / маркировка

`Скотч / ленты / маркировка` — отдельная подкатегория расходников для монтажных, маркировочных и временных задач на площадке.

Стартовые типы:

- gaffer tape;
- изолента;
- сигнальная лента;
- маркировочная лента;
- двусторонний скотч;
- малярная лента;
- прочее.

Минимальные поля карточки:

- тип;
- цвет;
- длина / рулон;
- количество;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Эта подкатегория нужна для закупки, комплектации, маркировки кабелей, временной фиксации, обозначения зон и других монтажных задач.

Клиенту эти позиции по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Кабельные стяжки

`Кабельные стяжки` — отдельная простая подкатегория расходников для крепления, группировки и временной фиксации кабелей и мелких элементов на площадке.

Минимальные поля карточки:

- длина;
- ширина;
- цвет;
- материал / тип: обычная / усиленная / многоразовая / металлическая / другое;
- упаковка / количество в пачке;
- количество пачек;
- цена клиенту;
- внутренняя себестоимость;
- комментарий.

Эта карточка нужна, чтобы не терять важные параметры стяжек: размер, цвет, тип и фасовку.

Клиенту кабельные стяжки по умолчанию не раскрываются отдельно и попадают в общий блок `Расходные материалы`, если менеджер специально не решил показать детализацию.

## Внутреннее использование

Подробный список расходников нужен для:

- закупки;
- склада;
- комплектации;
- списания;
- загрузки;
- технических листов;
- внутренней себестоимости проекта;
- контроля маржи.

## Отображение клиенту

По умолчанию клиент видит только общий блок расходников.

Логика:

```text
Подробный список расходников в проекте
→ внутренние технические, складские и закупочные листы
→ одна итоговая строка в клиентской смете
```

Если менеджер хочет раскрыть отдельный расходник клиенту, это должно быть ручным решением на уровне позиции или блока.

## Скидка

Расходники по умолчанию исключаются из базы скидки.

Причина: у расходников часто есть прямая закупочная или списываемая себестоимость, и автоматическая скидка может съедать маржу или создавать убыток.
