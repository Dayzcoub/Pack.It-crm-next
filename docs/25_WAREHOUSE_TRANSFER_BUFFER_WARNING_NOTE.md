# 25 Warehouse — Transfer Buffer Warning Note

## 10.143. Буфер времени для переезда оборудования между проектами

Решение: буфер времени для переезда оборудования между проектами должен оставаться на усмотрение менеджера, но система должна показывать предупреждение о риске.

Правило:
- система не должна жёстко блокировать использование одного комплекта в двух проектах за один день, если временные окна не пересекаются;
- менеджер может принять решение, что переезд реалистичен;
- если времени между окончанием первого проекта и стартом второго мало, система показывает предупреждение;
- предупреждение не запрещает сохранить проект или резерв;
- решение остаётся за менеджером.

Что учитывать в предупреждении:
- окончание первого проекта;
- старт второго проекта;
- демонтаж;
- погрузку;
- дорогу;
- разгрузку;
- монтаж;
- возможный запас времени.

Сценарий:
- проект A: 11:00–16:00;
- проект B: 17:00–22:00;
- один комплект RCF переезжает с проекта A на проект B;
- система видит цепочку использования и предупреждает, что между проектами всего 60 минут.

MVP-подход:
- не вводить обязательный жёсткий буфер;
- показывать warning при малом интервале между проектами;
- сохранить тип строки `transfer from another project`;
- дать менеджеру возможность продолжить, если он считает переезд реальным;
- позже можно добавить настраиваемый рекомендуемый буфер в админ-настройках.
