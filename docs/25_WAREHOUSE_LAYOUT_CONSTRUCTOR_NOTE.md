# 25.x. Конструктор плана склада

Этот документ дополняет складскую архитектуру PACK.IT CRM Next и фиксирует идею, возникшую после решения 10.51.

## Идея

В систему стоит заложить перспективный модуль конструктора плана склада.

Это визуальный инструмент, где пользователь может нарисовать площадь склада, расставить стеллажи, указать количество полок и назвать зоны хранения так, как удобно компании.

## Зачем это нужно

Такой модуль может сильно улучшить работу со складом:

- быстрее находить оборудование;
- понятнее показывать место хранения;
- упростить инвентаризацию по зонам;
- уменьшить путаницу со стеллажами, полками и ячейками;
- дать новым сотрудникам наглядную карту склада;
- связать карточку позиции с конкретным местом на плане.

## Базовая логика

Пользователь создаёт складскую локацию и может настроить:

- форму / площадь склада;
- зоны;
- стеллажи;
- количество полок;
- ячейки или места хранения;
- собственные названия зон и полок;
- комментарии.

Каждая позиция или экземпляр может быть привязан к месту на плане.

## Связь с инвентаризацией

Конструктор плана склада особенно полезен для инвентаризации.

Можно запускать проверку:

- по зоне;
- по стеллажу;
- по полке;
- по выбранным местам на плане.

Это позволит проводить частичную инвентаризацию быстрее и понятнее.

## MVP-статус

Для первого MVP конструктор плана склада не является обязательным ядром.

Но идею стоит заложить в архитектуру как будущий модуль `visual warehouse layout`.

На старте достаточно структурированных полей места хранения:

- склад / локация;
- зона;
- стеллаж;
- полка / ячейка;
- комментарий.

Позже эти поля можно связать с визуальным планом склада без полной переделки модели данных.

## Вывод

Идея хорошая и перспективная, особенно если PACK.IT должен вырасти из простого учёта склада в удобную операционную систему для технического проката.

Главное — не делать этот модуль обязательным для старта, но заранее проектировать место хранения так, чтобы визуальный план можно было подключить позже.
