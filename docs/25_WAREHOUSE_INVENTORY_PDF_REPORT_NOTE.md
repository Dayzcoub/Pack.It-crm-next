# 25.x. PDF-отчёт по итогам инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_SUMMARY_REPORT_NOTE.md` и фиксирует решение 10.91.

## Решение

После завершения инвентаризации нужно формировать PDF / печатный отчёт.

Экранная сводка остаётся в системе, но дополнительно должен быть файл, который можно сохранить, распечатать или отправить.

## Что включать в отчёт

PDF-отчёт может содержать:

- название компании;
- дату и время инвентаризации;
- кто проводил проверку;
- область проверки: склад, зона, стеллаж, полка или категория;
- общее количество ожидаемых позиций;
- сколько проверено;
- сколько найдено;
- сколько не найдено;
- сколько найдено не на своём месте;
- сколько создано проблемных строк;
- список проблемных строк;
- комментарий, если он указан.

## Назначение

PDF нужен как внутренний документ по итогам проверки.

Он не является финансовым отчётом и не должен превращать складскую инвентаризацию в бухгалтерский учёт.

## MVP

Для MVP достаточно простого PDF:

1. шапка проверки;
2. компактная сводка;
3. список проблемных строк;
4. дата формирования;
5. кто сформировал отчёт.

Детальный красивый шаблон можно улучшать позже.
