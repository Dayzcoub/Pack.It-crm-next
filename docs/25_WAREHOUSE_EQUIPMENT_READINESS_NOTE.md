# 25.x. Готовность оборудования и комплектность

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.34.

## Решение

Для оборудования не вводим отдельный статус `минимальная готовность` или отдельную схему `готово / не готово`.

Достаточно уже описанной комплектности оборудования.

Если у позиции есть шаблон комплекта, система может проверять его при сборке, выдаче и возврате.

Если комплект полный, позиция считается пригодной к нормальной выдаче.

Если комплект неполный или есть проблема, это фиксируется через отметки сборочного листа, проблемные строки возврата, комментарии и состояние экземпляра.

Отдельный статус готовности можно добавить позже, если складская логика станет сложнее, но для MVP это лишняя сущность.
