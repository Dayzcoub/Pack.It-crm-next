# 25.x. Массовая смена места хранения через сканер

Этот документ дополняет `docs/25_WAREHOUSE_SCANNER_CHANGE_STORAGE_PLACE_NOTE.md` и фиксирует решение 10.81.

## Решение

Массовую смену места хранения через мобильный сканер не делаем.

Место хранения нужно менять по одному объекту за раз.

## Почему так

Массовый режим может ускорить работу, но повышает риск ошибочно привязать много позиций к неправильной полке, зоне или стеллажу.

Для складского места хранения важнее точность, чем максимальная скорость.

## Правило

Пользователь сканирует один объект, выбирает новое место хранения, видит подтверждение и сохраняет изменение.

После этого можно перейти к следующему объекту.

## MVP

Для MVP поддерживаем только поштучную смену места хранения:

1. скан объекта;
2. выбор нового места;
3. подтверждение;
4. сохранение изменения;
5. запись в историю.

Массовый режим можно не закладывать в стартовую реализацию.
