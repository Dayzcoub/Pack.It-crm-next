# 25 Warehouse — Picking List By Category Note

## 10.136. Сборочный лист по проекту и категориям

Решение: сборочный лист должен поддерживать разделение по категориям / направлениям.

Правило:
- общий сборочный лист проекта нужен;
- дополнительно должна быть возможность сформировать отдельные сборочные листы по категориям или направлениям;
- категории зависят от структуры склада и проекта.

Примеры отдельных листов:
- звук;
- свет;
- LED;
- сцена;
- фермы;
- коммутация;
- питание;
- инструменты;
- кейсы / контейнеры.

Назначение:
- каждый профильный специалист может получить свой понятный список;
- склад может собирать проект по зонам ответственности;
- большой проект не превращается в один огромный нечитаемый документ.

MVP-подход:
- формировать общий сборочный лист;
- добавить фильтр/группировку по категории;
- PDF/печать может быть общий или по выбранной категории;
- цены не выводить, сборочный лист остаётся внутренним документом.
