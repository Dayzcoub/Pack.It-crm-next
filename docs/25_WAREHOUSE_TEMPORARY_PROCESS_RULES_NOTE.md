# 25 Warehouse — Temporary Process Rules Note

## 10.123. Временные правила склада

Решение: временные правила склада нужны как инструмент.

Сценарии:
- сезонная нагрузка;
- переезд склада;
- временное усиление контроля;
- разбор последствий проблемной ситуации;
- тестирование более строгого процесса перед постоянным включением.

Правило:
- временное правило не должно быть обязательным механизмом для всех;
- администратор может включить временное правило для конкретного складского процесса;
- у временного правила должна быть дата окончания или ручное отключение;
- после окончания временного периода система возвращает прежнее правило;
- временное правило должно быть видно в административных настройках склада.

Что хранить:
- процесс;
- правило;
- временное значение;
- исходное значение;
- дата начала;
- дата окончания, если задана;
- кто включил;
- комментарий причины;
- статус временного правила.

Архитектурный смысл:
- временные правила позволяют усилить складской процесс без постоянной перенастройки;
- компания может аккуратно проверять более строгие правила;
- после завершения периода склад возвращается к обычному режиму.

MVP-подход:
- заложить поддержку временных правил как опциональный административный инструмент;
- не использовать временные правила в базовом сценарии маленького склада;
- на старте достаточно ручного включения и даты окончания;
- автоматическое возвращение к исходному значению желательно предусмотреть в модели.
