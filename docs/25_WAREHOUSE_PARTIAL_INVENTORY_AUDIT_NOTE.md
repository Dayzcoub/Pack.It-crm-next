# 25.x. Частичная и профильная инвентаризация

Этот документ дополняет `docs/25_WAREHOUSE_INVENTORY_AUDIT_NOTE.md` и фиксирует решение 10.50.

## Решение

Инвентаризацию нужно уметь запускать не только по всему складу, но и частично.

Это важно для маленьких прокатов, где инвентаризацию могут проводить не отдельные складские работники, а профильные специалисты по своим зонам ответственности.

Например, один специалист проверяет свои категории оборудования, другой специалист — свои.

## Варианты области инвентаризации

Система должна поддерживать запуск инвентаризации по области:

- весь склад;
- категория;
- подкатегория;
- зона хранения;
- стеллаж / полка / ячейка;
- выбранный список позиций;
- позиции, закреплённые за ответственным или ролью, если такая настройка включена.

## Логика для маленьких компаний

Для маленьких команд важно, чтобы инвентаризация не требовала отдельного складского отдела.

Профильный специалист может запустить проверку только по тем позициям, за которые он отвечает или которые относятся к его направлению.

Такой подход позволяет проводить инвентаризацию постепенно, без остановки всей работы склада.

## Логика для больших складов

Для больших складов частичная инвентаризация также полезна:

- можно проверять зоны по графику;
- можно проверять категории отдельно;
- можно назначать разные проверки разным сотрудникам;
- можно быстрее закрывать проблемные участки.

## MVP

Для MVP достаточно:

1. при создании инвентаризации выбрать область проверки;
2. показать список позиций только из выбранной области;
3. дать пользователю отметить результаты;
4. сохранить итоги и проблемные строки.

Полная инвентаризация всего склада остаётся возможной, но не должна быть единственным вариантом.
