# 25.x. Права запуска инвентаризации

Этот документ дополняет `docs/25_WAREHOUSE_PARTIAL_INVENTORY_AUDIT_NOTE.md` и фиксирует решение 10.51.

## Решение

Инвентаризацию должны иметь возможность запускать не только администратор или директор.

Профильные сотрудники могут запускать инвентаризацию самостоятельно по своим категориям, зонам ответственности или доступным им складским областям.

Это важно для маленьких прокатов, где нет отдельного складского отдела, а оборудование по направлениям проверяют профильные специалисты.

## Логика доступа

Администратор и директор могут запускать инвентаризацию по всему складу и любым областям.

Профильный сотрудник может запускать проверку только по тем областям, которые доступны ему по роли, категории, назначенной ответственности или настройкам прав.

Складской работник может запускать инвентаризацию в рамках своих складских прав.

## Ограничения

Запуск инвентаризации не должен давать пользователю дополнительных прав на:

- списание;
- изменение закупочной стоимости;
- изменение прав доступа;
- удаление складских позиций;
- подтверждение критичных складских действий.

Пользователь может провести проверку и зафиксировать результат, но проблемные действия по итогам проверки должны обрабатываться согласно отдельным правилам доступа.

## MVP

Для MVP достаточно:

1. пользователь выбирает доступную ему область инвентаризации;
2. система показывает только разрешённые позиции;
3. пользователь отмечает результаты проверки;
4. проблемные строки сохраняются для дальнейшего разбора.
