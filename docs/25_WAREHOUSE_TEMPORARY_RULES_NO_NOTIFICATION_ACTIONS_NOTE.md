# 25 Warehouse — Temporary Rules No Notification Actions Note

## 10.127. Действия продления из уведомления о временном правиле

Решение: продлевать временное правило прямо из уведомления не нужно.

Правило:
- уведомление только сообщает, что временное правило скоро закончится;
- быстрых кнопок `продлить на неделю` или `изменить дату окончания` в уведомлении не делаем;
- изменение периода выполняется отдельно в административных настройках склада;
- уведомление может вести в карточку/настройки временного правила, но не должно превращаться в отдельный мини-интерфейс управления.

Причина:
- не перегружать уведомления лишними действиями;
- настройка временных правил должна оставаться в одном понятном месте;
- админ должен осознанно открыть настройки и поменять срок, если это действительно нужно.

MVP-подход:
- уведомление за 1 день до окончания;
- без action buttons для продления;
- максимум — ссылка на административные настройки временных правил.
