# 25.x. Формат визуального плана склада

Этот документ дополняет `docs/25_WAREHOUSE_LAYOUT_CONSTRUCTOR_NOTE.md` и фиксирует решение 10.52.

## Решение

Для конструктора плана склада достаточно 2D-вида сверху.

Полноценный 3D-вид стеллажей и полок для MVP не нужен.

При этом нужно заложить поддержку этажности / уровней хранения, чтобы один и тот же стеллаж или зона могли иметь несколько уровней, полок или ячеек.

## Базовая модель

План склада отображается сверху как 2D-схема.

На схеме можно размещать:

- зоны;
- стеллажи;
- полки / уровни;
- ячейки;
- проходы;
- служебные области;
- комментарии.

Каждый объект на плане может иметь название и внутренний код.

## Этажность и уровни

Для стеллажей и мест хранения нужно поддерживать уровни.

Пример структуры:

- склад;
- зона;
- стеллаж;
- полка / уровень;
- ячейка;
- комментарий.

Пользователь видит 2D-план сверху, а уровни выбирает внутри карточки стеллажа, зоны или места хранения.

## Почему без 3D

3D-визуализация склада может усложнить интерфейс и разработку, но не является обязательной для основной задачи.

Основная задача — быстро понимать, где находится позиция, и удобно запускать инвентаризацию по зоне, стеллажу, полке или ячейке.

2D-плана с этажностью достаточно для этого сценария.

## MVP

Для MVP достаточно:

1. 2D-план сверху;
2. объекты склада на плане;
3. настройка названий зон и стеллажей;
4. количество уровней / полок;
5. привязка позиции или экземпляра к месту хранения;
6. запуск инвентаризации по выбранному месту.

Полноценный 3D-режим можно не закладывать как обязательный модуль.
