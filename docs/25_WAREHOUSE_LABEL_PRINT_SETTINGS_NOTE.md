# 25.x. Настройка содержимого QR и штрихкод-наклеек

Этот документ дополняет `docs/25_WAREHOUSE_MACHINE_MARKS_NOTE.md` и фиксирует решение 10.67.

## Решение

Содержимое QR / штрихкод-наклеек для оборудования, экземпляров и кейсов должно настраиваться администратором.

Не нужно жёстко задавать один формат наклейки для всех компаний.

## Логика

Администратор может выбрать, какие данные печатать на наклейке.

Например:

- внутренний код;
- название позиции;
- номер экземпляра;
- серийный номер;
- QR-код;
- штрихкод;
- логотип компании;
- короткий комментарий;
- название складской локации или зоны, если нужно.

## Пример

```text
SND-000245
Shure QLXD BETA58
Экз. #003
QR-код
```

## Почему так

У разных компаний разные привычки маркировки.

Одним достаточно QR-кода и внутреннего кода.

Другим нужно видеть название позиции и номер экземпляра прямо на наклейке.

Настройка в админке позволит не менять логику системы под каждую компанию.

## MVP

Для MVP достаточно одного или нескольких шаблонов наклейки и настройки, какие поля показывать.

Печать внутреннего кода на наклейке должна быть доступна как настройка, но не обязательно быть единственным вариантом.
