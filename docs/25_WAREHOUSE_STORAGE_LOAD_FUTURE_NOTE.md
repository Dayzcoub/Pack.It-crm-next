# 25.x. Загруженность мест хранения

Этот документ дополняет `docs/25_WAREHOUSE_LAYOUT_CONSTRUCTOR_NOTE.md` и фиксирует решение 10.56.

## Решение

Загруженность полок, стеллажей и мест хранения пока не делаем в MVP.

При этом в фундамент архитектуры нужно заложить возможность добавить такую функцию позже.

## Как заложить фундамент

В модели места хранения можно предусмотреть будущие параметры вместимости, но не выводить их как обязательный функционал.

Например, в будущем место хранения сможет иметь:

- вместимость;
- условные единицы хранения;
- расчёт занятого объёма;
- процент загрузки;
- предупреждение о перегрузе.

На старте эти параметры можно не использовать.

## Почему не делать сейчас

Сейчас важнее сделать:

- понятное место хранения;
- зоны, стеллажи, полки и уровни;
- поиск с подсветкой на плане;
- инвентаризацию по месту хранения;
- простую привязку позиций к складу.

Загруженность полок можно продумать в следующих итерациях, когда станет понятнее реальная эксплуатация склада.

## MVP

Для MVP не показываем:

- процент занятости полки;
- свободный объём;
- перегруз стеллажа;
- автоматический расчёт вместимости.

Но структура места хранения должна быть такой, чтобы позже можно было добавить расчёт загруженности без переделки всей модели склада.
