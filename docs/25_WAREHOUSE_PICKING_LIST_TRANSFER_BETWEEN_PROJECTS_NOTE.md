# 25 Warehouse — Picking List Transfer Between Projects Note

## 10.142. Добор, следующий заезд и переезд оборудования между проектами

Решение: кроме обычного добора, складские документы должны учитывать сценарий переезда оборудования между проектами в один день.

Сценарий:
- в один день есть две площадки;
- первая работает, например, с 11:00 до 16:00;
- вторая работает с 17:00 до 22:00;
- на обеих площадках нужны одинаковые позиции;
- часть оборудования есть в достаточном количестве, например 12 радиомикрофонов для двух площадок по 6;
- часть оборудования есть только в одном комплекте, например один комплект звука RCF;
- комплект RCF должен быть сначала выдан на первую площадку, затем переехать на вторую.

Вывод:
- `добор` не должен означать только недостающие позиции со склада;
- нужен отдельный логистический сценарий `переезд / transfer между проектами`;
- позиция может быть доступна для второго проекта не со склада, а после возврата/демонтажа с первого проекта;
- availability должна учитывать не только дату, но и временные окна проектов.

Правило:
- если оборудование используется в двух проектах в один день с непересекающимися временными окнами, система может разрешить цепочку использования;
- нужно учитывать буфер на демонтаж, погрузку, дорогу, разгрузку и монтаж;
- если буфера не хватает, система должна предупреждать о риске;
- переезд между проектами должен отображаться отдельным списком или блоком, связанным с обоими проектами.

Документы:
- основной сборочный лист проекта остаётся главным;
- `Добор` может использоваться для недостающих позиций, которые довезут позже;
- для оборудования, которое переезжает с другой площадки, лучше использовать отдельный блок/лист `Переезд / transfer`;
- transfer-лист показывает, что забрать с первой площадки и привезти на вторую.

MVP-подход:
- заложить модель time-window availability;
- заложить связь позиции с несколькими проектами в один день, если временные окна позволяют;
- заложить статус/тип строки: from warehouse / dobor next trip / transfer from another project;
- на старте можно показывать это как предупреждение и отдельный блок в сборочном листе, без сложной оптимизации маршрутов.
