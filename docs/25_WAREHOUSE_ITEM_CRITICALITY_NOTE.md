# 25.x. Критичность складской позиции

Этот документ дополняет `docs/25_WAREHOUSE_AND_INVENTORY.md` и фиксирует решение 10.43.

## Решение

Категорию критичности складской позиции можно заложить в архитектуру, но не делать на ней акцент.

Для текущих реалий это не выглядит обязательной функцией, поэтому поле не должно перегружать карточку позиции и складской интерфейс.

Подход для MVP:

- поле критичности опциональное;
- по умолчанию скрыто или второстепенно;
- не обязательно для заполнения;
- не должно блокировать выдачу, сборку или возврат без отдельной настройки;
- может использоваться в будущем для фильтров, предупреждений или расширенного складского контроля.

Если функция понадобится позже, можно добавить значения критичности и связанные правила, но базовый склад должен работать без этого поля.
