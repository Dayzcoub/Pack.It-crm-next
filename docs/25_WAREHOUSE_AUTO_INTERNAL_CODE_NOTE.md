# 25.x. Автоматическая генерация внутреннего кода позиции

Этот документ дополняет `docs/25_WAREHOUSE_ITEM_CODES_NOTE.md` и фиксирует решение 10.63.

## Решение

Внутренний код складской позиции должен генерироваться автоматически.

Пользователь не должен вручную придумывать системный код для каждой позиции.

## Почему так

Автоматическая генерация помогает:

- не плодить разнобой в кодах;
- быстрее заводить позиции;
- сохранять единый стиль каталога;
- уменьшать ошибки при вводе;
- использовать код как стабильный внутренний идентификатор.

## Логика

Система может генерировать код на основе:

- категории;
- подкатегории;
- бренда;
- модели;
- типа позиции;
- порядкового номера;
- артикула производителя, если он указан.

## Редактирование

Для MVP можно сделать код автоматическим и доступным для ручного исправления только пользователям с расширенными правами.

Обычный пользователь склада или менеджер не должен быть обязан редактировать код вручную.

## Правило MVP

Если пользователь создаёт новую позицию, система сама присваивает внутренний код.

Если данных мало, система всё равно создаёт технический уникальный код.

Код должен быть уникальным внутри компании.
